# ARC.ONE 落地盘点

## 已验证基础

- Agent 草稿、编辑、版本发布、历史、停用和测试运行。
- 工作流草稿、DAG 校验、Agent 版本引用、发布与顺序执行。
- Run、NodeRun、Artifact、Human Task 和审核决定持久化。
- Human 节点暂停、会签、恢复、重跑、终止和反馈候选。
- DeepSeek OpenAI-compatible 真实调用已验证。
- 前端 30 项测试、后端 32 项测试、Lint、Build 和浏览器路径已有记录。

## 主要缺口

- 评估中心仍是展示，缺少 Rubric 编辑、评价器、Judge 和回归执行。
- 工作流缺少变量映射、并行汇聚、条件路由、子流程和断点恢复。
- 缺少可靠异步 Worker、实时事件、后台 SLA 扫描和外部通知消费。
- Tool、Skill、MCP 仍是配置字段，不是独立资产与受控执行能力。
- 缺少登录、组织、RBAC、密钥管理、环境隔离和预算配额。
- 缺少完整 Trace、日志查询、成本口径、告警和运行回放。
- PostgreSQL、正式迁移、Docker、CI/CD、备份恢复和生产部署未验证。
- 尚未选定并量化一条真实业务试点。

## V0.7 已确认决策

- 首版使用平台本地账号与登录会话，预留 OIDC/飞书 SSO 接口。
- 采用单组织、多 Workspace、多成员模型，数据预留组织归属。
- 使用固定平台角色：组织管理员、Workspace 管理员、构建者、运行者、观察者。
- Reviewer 是绑定用户的审核资格，不是独立账号或平台角色。
- 本地认证采用邮箱和密码、服务端 HttpOnly Session Cookie。
- 首位组织管理员通过命令创建，其他成员通过一次性邀请链接激活。
- 升级时创建默认 Workspace `AI 能力中心`，现有资产统一迁入。
- 所有新资产必须绑定 Workspace，V0.7 不支持跨 Workspace 共享、移动或复制。
- Workspace 角色权限逐级包含：观察者只读；运行者可运行；构建者可创建、编辑和发布；
  Workspace 管理员可停用资产并管理成员与审核资格。
- 任意平台角色只有获得审核资格后才能处理 Human Task。
- 组织管理员可创建 Workspace 并管理组织级成员和治理策略。
- 权限必须在后端 API 强制执行，前端按钮可见性不能作为安全边界。
- 审计覆盖认证、成员与角色、Workspace、资产生命周期、运行、人工审核、
  审核资格和权限拒绝。
- 审计记录允许查询和导出，不允许修改或删除。
- V0.7 拆为两个可独立验收版本：
  - V0.7A：登录、会话、组织、Workspace、成员、RBAC、Reviewer 用户绑定和审计。
  - V0.7B：密钥托管、环境隔离、预算配额、模型/工具白名单和安全策略。
- Session 连续 8 小时有效，活动时滑动续期，绝对最长 7 天。
- 修改密码或停用账号后撤销该用户全部 Session。
- 邀请链接 72 小时有效且只能使用一次。
- 连续 5 次登录失败锁定 15 分钟。
- Cookie 使用 HttpOnly 与 SameSite=Lax，生产 HTTPS 强制 Secure。
- 所有写请求具备 CSRF 防护；Session 撤销、登录限流和 CSRF 属于 V0.7A 验收范围。
- Web 路由显式使用 `/w/{workspaceSlug}/...`，API 显式使用
  `/api/workspaces/{workspaceId}/...`，不在 Cookie 中隐藏当前 Workspace。
- 登录后进入最近访问的 Workspace；无权限返回 403，资源不存在或跨 Workspace
  探测统一返回 404。
- 成员管理支持邀请、重发、撤销邀请、改角色、启停成员和审核资格管理。
- 禁止管理员停用自己，每个 Workspace 至少保留一名 Workspace 管理员。
- V0.7A 不彻底删除成员，不做批量导入和组织架构同步；停用后仍保留历史引用。
- 现有 Reviewer 保留原 ID 与历史关联，并绑定新建的待激活 User。
- 正式迁移不生成默认密码；管理员补录邮箱后发送一次性激活邀请。
- 未激活 Reviewer 可保留历史展示，但不能认领或处理新任务。
- 开发演示数据允许通过初始化命令创建已激活测试用户。
- V0.7A 安全测试必须覆盖未登录、错误密码与锁定、Session 失效、CSRF、
  五种角色权限、跨 Workspace 隔离、邀请边界、最后管理员保护、Reviewer 资格、
  停用用户旧 Session 和审计完整性。
- 浏览器路径覆盖登录、Workspace 切换、邀请成员、角色变化和退出。
- V0.7A 不发送邀请邮件；管理员复制一次性激活链接并通过现有企业渠道发送。
- 明文邀请令牌只在创建或重发成功响应中返回一次，复制动作进入审计。

## V0.7A 实施文件边界

- 现有 FastAPI 路由集中在 `apps/api/app/main.py`，V0.7A 不继续把认证和授权规则
  直接堆入该文件。
- 新增独立认证、访问控制和审计服务模块，由 `main.py` 负责依赖装配与路由注册。
- 现有 `HumanTaskService` 保留人工任务业务规则，但操作者改由请求上下文传入，
  不再接受前端伪造 reviewerId。
- 前端现有 API 统一经过 `src/api/http.ts`，V0.7A 在该入口集中处理 Cookie、CSRF、
  401 会话失效和 Workspace 路径。
