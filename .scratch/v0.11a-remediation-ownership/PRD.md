# PRD：V0.11A 修复任务负责人、截止时间和优先级筛选

## 背景

V0.10 已经形成“失败原因 -> 修复任务 -> 复测 -> 闭环看板”的评估闭环。下一步需要让修复任务进入可运营状态：任务要能明确负责人、截止时间和优先级，并支持按这些字段筛选。

## 目标

- Remediation Task 持久化负责人和截止时间。
- 创建任务时可带默认负责人和截止时间。
- 后端任务列表支持按负责人、优先级和逾期状态筛选。
- 前端 Remediation Tasks 区域展示负责人、截止时间和逾期状态。
- 前端提供负责人、优先级和逾期筛选。

## 非目标

- 不做完整任务评论时间线。
- 不做附件上传。
- 不做飞书外部通知。
- 不做完整个人工作台。

## 用户故事

作为评估中心管理员，我希望失败修复任务能看到负责人和截止日期，这样我能知道谁该处理、哪些任务已经逾期。

作为修复负责人，我希望能按“我的任务”或优先级过滤任务，这样我不用在所有任务中手动找。

## 验收标准

- 创建 Remediation Task 时后端保存 `owner` 和 `dueDate`。
- 历史 SQLite 表可增量补列，不破坏旧数据。
- `GET /remediation-tasks` 支持 `owner`、`priority`、`overdue` 查询参数。
- 前端任务卡展示负责人、截止时间和逾期/未逾期状态。
- 前端 Remediation Tasks 区域支持负责人、优先级和逾期筛选。
- 原有任务创建、状态流转、复测和闭环看板不回归。
