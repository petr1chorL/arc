# V0.10E Regression Run 质量洞察摘要 PRD

## 问题陈述

V0.10D 已经能展示 Regression Run 趋势，但用户仍需要自己阅读指标和柱状图，判断最近质量是否稳定、变差或需要处理。

## 解决方案

在 `Regression Run Trend` 内新增 `Regression Run Insight` 摘要卡。它基于当前筛选后的 Run 趋势自动给出质量状态、关键事实和下一步建议。

## 用户故事

- 作为质量负责人，我希望看到一句明确的质量判断，而不是只看通过率数字。
- 作为工作流构建者，我希望快速知道最近一次回归是否比上次变差。
- 作为运营人员，我希望当最近 Run 低于风险线时能看到可执行建议。

## 实施决策

- 只做前端轻量洞察，不新增后端聚合 API。
- 洞察基于当前筛选后的最近最多 8 次 Regression Run。
- 风险线沿用 V0.10D：通过率低于 70%。
- 输出包含：状态标题、事实摘要、建议、风险 Run 数。

## 测试决策

- 使用 `src/pages/Evaluations.test.tsx` 增加红灯测试。
- 测试用三次 Run 构造最新通过率下滑且低于风险线的场景。
- 验证页面出现 `Regression Run Insight`、质量下滑标题、最新通过率、较上次变化、风险 Run 数和建议文案。

## 范围外

- 不做 LLM 生成洞察。
- 不做后端趋势聚合。
- 不做阈值配置。
- 不做导出报告。
