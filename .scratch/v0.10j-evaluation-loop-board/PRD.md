# V0.10J PRD：评估闭环看板

## 问题陈述

V0.10H 和 V0.10I 已经完成“失败原因 -> 修复任务 -> 复测 Run”的链路，但用户仍需要在页面多个区块之间来回看，才能判断当前评估闭环是否健康。

## 解决方案

在评估中心增加 `Evaluation Loop Board`，把关键闭环指标集中展示：

- 当前失败原因组数量。
- Remediation Task 总数。
- 未关闭风险数。
- 已复测任务数。
- 最近复测通过率。
- 下一步建议。

## 用户故事

作为质量负责人，我希望打开评估中心时快速知道当前还有多少未关闭质量风险，以及最近复测是否有效。

作为平台管理员，我希望闭环看板能把失败原因、修复任务和复测结果串起来，而不是只展示孤立列表。

## 实施决策

- 仅做前端派生看板，不新增后端 API。
- 数据来源：
  - `failurePatternSummary.clusters`：失败原因组。
  - `remediationTasks`：任务数量、未关闭风险、复测任务。
  - `remediationTasks[].retestRun`：最近复测通过率和失败数。
- 未关闭风险定义：任务状态不是 `done`，或任务已 `done` 但还没有 `retestRunId`。
- 下一步建议由确定性规则生成。

## 测试决策

- 通过 `src/pages/Evaluations.test.tsx` 覆盖有失败原因和复测任务时的看板指标。
- 不单独引入图表库。

## 范围外

- 不做后端统计 API。
- 不做时间序列趋势图。
- 不做负责人维度分析。
- 不做告警通知。
