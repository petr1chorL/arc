# V0.11C 复测失败回流 PRD

## 背景

V0.11B 已经支持修复任务负责人、截止时间、评论和处理时间线。当前缺口是：修复任务被人工标记完成后，如果复测仍然失败，任务仍可能显示为完成态，导致闭环看板误以为风险已经关闭。

## 目标

当修复任务复测失败时，系统自动把任务回流到处理中，并在时间线中留下明确记录，保证失败问题不会被误报为完成。

## 范围

- 复测失败后，原任务从 `done` 自动回到 `in_progress`。
- 复测失败后，任务时间线新增复测失败记录。
- 复测失败后，任务时间线新增状态变更记录。
- 任务再次标记完成后，可以重新发起新一轮复测。
- 前端任务卡展示“复测失败已回流”状态。
- Evaluation Loop Board 把回流后的任务计为未关闭风险。

## 不做

- 不新增子任务模型。
- 不接入外部通知。
- 不改变 Regression Run 的评分逻辑。
- 不引入后台队列。

## 验收标准

1. 未完成任务发起复测仍返回 409。
2. 已完成任务发起复测后，如果复测失败，任务状态返回 `in_progress`。
3. 失败复测 Run 仍写入 `retestRunId` 和 `retestRun`。
4. 任务 activities 包含 `retest_failed` 和 `status_change`。
5. 失败回流任务再次标记完成时，会清理旧的失败复测引用，允许重新发起复测。
6. 前端复测失败后显示“复测失败已回流”。
7. Evaluation Loop Board 显示未关闭风险数量大于 0。
