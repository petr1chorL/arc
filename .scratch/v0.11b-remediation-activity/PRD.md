# PRD：V0.11B 修复任务评论与处理记录

## 背景

V0.11A 已经让 Remediation Task 具备负责人、截止时间和筛选能力，但任务仍然缺少“处理过程”。当任务被标记完成或延期时，平台需要能解释为什么发生这些变化，并保留人工处理记录。

## 目标

- Remediation Task 支持评论。
- 评论支持附件引用字符串，不做真实上传。
- Remediation Task 状态变化自动形成处理记录。
- 前端任务卡展示处理时间线。
- 前端支持输入评论和附件引用。

## 非目标

- 不做真实附件上传。
- 不做富文本编辑器。
- 不做外部通知。
- 不做任务评论权限细分；沿用现有修复任务更新权限。

## 用户故事

作为任务负责人，我希望能在修复任务上记录处理说明和附件引用，这样别人能看到我做了什么。

作为评估中心管理员，我希望任务状态变化自动进入时间线，这样我能追溯任务为什么从待处理变成处理中或已完成。

## 验收标准

- 后端新增 Remediation Task 活动记录持久化。
- `POST /remediation-tasks/{taskId}/activities` 可创建评论。
- 评论返回正文、附件引用、操作者和时间。
- PATCH 更新任务状态时自动写入状态变化记录。
- 任务列表返回每个任务的活动记录。
- 前端任务卡展示时间线。
- 前端可提交评论和附件引用。
- 原有任务创建、筛选、状态流转、复测和闭环看板不回归。
