# ARC.ONE 从当前版本到 V1.0 的落地步骤

> 更新时间：2026-06-27  
> 用途：回答“后面还有多少步”，作为后续版本推进和验收的主索引。

## 总体判断

从当前 V0.11D 到 V1.0，建议继续按 5 个阶段拆分推进。每个小版本都必须能单独验收，不把规划能力说成已实现能力。

## 阶段 1：收敛评估闭环，完成 V0.10

目标：让评估中心从“看分数”进入“发现问题、生成修复任务、复测验证”的闭环。

1. V0.10H：失败修复任务。
   - 从失败原因队列创建可追踪任务。
   - 支持任务状态 `open -> in_progress -> done`。
   - 验收：创建任务、状态流转、重复创建去重。
   - 当前状态：已完成。
2. V0.10I：任务关联复测。
   - 已完成任务可选择样本一键发起 Regression Run。
   - 验收：任务详情能看到复测 Run 和复测结果。
   - 当前状态：已完成。
3. V0.10J：评估闭环看板。
   - 汇总失败原因、修复任务、复测通过率和未关闭风险。
   - 验收：管理者能看出当前最该处理的问题。
   - 当前状态：已完成。

## 阶段 2：执行闭环与任务运营，V0.11

目标：让平台从“能运行工作流”进入“能运营执行过程”。

4. V0.11A：修复任务负责人和截止时间。
   - 增加 owner、due date、优先级筛选。
   - 验收：不同负责人能看到自己的修复任务。
   - 当前状态：已完成。
5. V0.11B：任务评论与处理记录。
   - 支持记录修复过程、变更说明和附件引用。
   - 验收：任务时间线能解释为什么完成或延期。
   - 当前状态：已完成。
6. V0.11C：复测失败回流。
   - 复测失败时自动重新打开任务或生成子任务。
   - 验收：复测未通过不会被误报为完成。
   - 当前状态：已完成。
7. V0.11D：执行事件流。
   - 将运行、人工审核、修复任务和复测事件统一成可查询事件。
   - 验收：从一个 Trace 能看到完整业务链路。
   - 当前状态：已完成。

## 阶段 3：真实 Agent 与工具能力，V0.12

目标：让 Agent 从“资产配置 + 模型调用”升级为“可组合、可治理、可扩展的执行单元”。

8. V0.12A：Agent Runtime 抽象。
   - 定义 Agent 输入、输出、错误、Token、成本和工具调用协议。
   - 验收：不同 Agent 节点按统一协议执行。
   - 当前状态：已完成。
9. V0.12B：Tool / Skill 资产库。
   - 独立管理工具、权限、参数 Schema 和调用日志。
   - 验收：Agent 只能调用已授权工具。
   - 当前状态：资产库与 Agent 绑定校验已完成；调用日志待做。
10. V0.12C：MCP / HTTP 工具适配。
   - 接入 MCP Server 或企业 HTTP API。
   - 验收：工具调用结果进入节点产出物和 Trace。
11. V0.12D：LLM-as-a-Judge。
   - Rubric 可选择确定性评分器或 LLM Judge。
   - 验收：评分结果带理由、模型、版本和可复现输入。

## 阶段 4：工作流调度与生产可靠性，V0.13

目标：让工作流从同步原型升级为可恢复、可观测、可治理的调度系统。

12. V0.13A：异步任务队列。
   - 引入后台执行队列，支持长任务不阻塞 API。
   - 候选工具：Celery / Dramatiq / RQ；后续生产可评估 Temporal。
13. V0.13B：工作流暂停、恢复、取消和重试。
   - 节点级重试、人工恢复、失败继续策略。
   - 验收：失败节点可定位、重试、恢复。
14. V0.13C：条件路由、并行和汇聚。
   - 支持分支表达式、并行节点、汇聚策略。
   - 验收：复杂 DAG 能正确执行并保存节点结果。
15. V0.13D：实时事件推送。
   - SSE 或 WebSocket 推送运行状态。
   - 验收：运行中心无需刷新即可更新。

## 阶段 5：企业化与 V1.0 发布

目标：满足真实组织使用的安全、部署、运维和试点要求。

16. V0.14A：组织、角色和权限硬化。
   - 完整 RBAC、成员审计、权限测试。
   - 验收：跨 Workspace 数据不可见，敏感操作有审计。
17. V0.14B：外部通知。
   - 消费 NotificationOutbox，接入飞书通知。
   - 验收：SLA 到期、任务分派、运行失败可通知到人。
18. V0.14C：PostgreSQL 与迁移体系。
   - 正式迁移工具、初始化脚本、备份恢复。
   - 验收：SQLite 开发数据与 PostgreSQL 生产环境路径清晰。
19. V0.14D：部署与环境管理。
   - Docker Compose、环境变量模板、启动/停止脚本。
   - 验收：新机器可按文档启动。
20. V0.14E：CI/CD 与质量门禁。
   - 自动运行测试、lint、build、迁移检查。
   - 验收：未通过门禁不能合并。
21. V0.15A：试点业务流。
   - 选 1-2 条真实业务流程落地。
   - 验收：真实用户能从输入到产出再到评估闭环跑通。
22. V1.0 RC：稳定性与性能压测。
   - 并发运行、失败恢复、日志与监控。
   - 验收：试点场景连续运行稳定。
23. V1.0：正式发布。
   - 冻结范围、交付部署包、验收手册和运维手册。
   - 验收：业务方按手册独立完成核心流程。

## 关键开源工具候选

- 前端：React、TypeScript、Vite、React Router、React Flow、Lucide。
- 服务端：FastAPI、Pydantic、SQLAlchemy。
- 数据库：SQLite 开发，PostgreSQL 生产。
- 测试：Vitest、Testing Library、Pytest、Playwright。
- 队列：Dramatiq / Celery / RQ，生产复杂编排可评估 Temporal。
- 观测：OpenTelemetry、Prometheus、Grafana、Loki。
- AI 接入：OpenAI-compatible API、MCP、后续可补 LiteLLM 或自研 ModelGateway 扩展。

## 当前推荐下一步

下一步继续 V0.12B：Tool / Skill 调用日志。原因是资产库和 Agent 绑定校验已经完成，但 Runtime 还没有把工具调用行为记录成可审计日志。
