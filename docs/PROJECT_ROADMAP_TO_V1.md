# ARC.ONE 从当前版本到 V1.0 的落地步骤

> 更新时间：2026-07-11
> 用途：回答“后面还有多少步”，作为后续版本推进和验收的主索引。

## 2026-06-29 路线调整：先做 V1.0 Lite

为了快速落地，当前路线从“大而全 V1.0”调整为“V1.0 Lite 真实业务试点版”。完整生产版仍保留为后续 V1.1+，当前优先证明一个真实业务流程能跑通：

```text
业务输入 -> Agent/Workflow -> 产出物 -> 人工审核 -> 质量评估 -> 运行观测 -> 问题闭环
```

详细范围、5-7 天计划和后置项见：

- `docs/V1_LITE_LAUNCH_PLAN.md`
- `docs/ACCEPTANCE_V1_LITE.md`
- `docs/V1_LITE_PILOT_PROCESS.md`

## 2026-07-11 当前阶段

V0.7A-V0.31F 的第一版实现已经进入 `master`，异步队列、身份权限、Evaluation、
Observability、Data Object、Artifact、Remediation 和 Notification Outbox 不再是下一步
待开发清单。当前阶段是 **V1.0 Lite 收口**：

1. **已完成：** 统一项目事实，避免归并前文档继续误导后续会话。
2. **已完成：** 恢复 Python 3.12、默认前端测试、标准 build 和登录 E2E，并取得同轮新证据。
3. 完成 P0 运行时安全人工签收和可能暴露模型 Key 的轮换。
4. 由真实业务方按手册独立完成一次端到端验收。
5. 根据试点结果决定可靠性主线或一个真实外部集成。

在上述收口完成前，不继续按细版本编号线性扩展页面或治理卡片。

## 总体判断

旧路线从 V0.11D 到 V1.0 分 5 个阶段推进。当前仓库已推进到 V0.31F，许多原阶段能力已经完成或被拆成更细切片。后续不再按旧编号线性补全所有增强项，而是优先服务 V1.0 Lite 试点落地。

## 阶段 1：收敛评估闭环，完成 V0.10

目标：让评估中心从“看分数”进入“发现问题、生成修复任务、复测验证”的闭环。

1. V0.10H：失败修复任务。
   - 从失败原因队列创建可追踪任务。
   - 支持任务状态 `open -> in_progress -> done`。
   - 验收：创建任务、状态流转、重复创建去重。
   - 当前状态：已完成。
2. V0.10I：任务关联复测。
   - 已完成任务可选择样本一键发起 Regression Run。
   - 验收：任务详情能看到复测 Run 和复测结果。
   - 当前状态：已完成。
3. V0.10J：评估闭环看板。
   - 汇总失败原因、修复任务、复测通过率和未关闭风险。
   - 验收：管理者能看出当前最该处理的问题。
   - 当前状态：已完成。

## 阶段 2：执行闭环与任务运营，V0.11

目标：让平台从“能运行工作流”进入“能运营执行过程”。

4. V0.11A：修复任务负责人和截止时间。
   - 增加 owner、due date、优先级筛选。
   - 验收：不同负责人能看到自己的修复任务。
   - 当前状态：已完成。
5. V0.11B：任务评论与处理记录。
   - 支持记录修复过程、变更说明和附件引用。
   - 验收：任务时间线能解释为什么完成或延期。
   - 当前状态：已完成。
6. V0.11C：复测失败回流。
   - 复测失败时自动重新打开任务或生成子任务。
   - 验收：复测未通过不会被误报为完成。
   - 当前状态：已完成。
7. V0.11D：执行事件流。
   - 将运行、人工审核、修复任务和复测事件统一成可查询事件。
   - 验收：从一个 Trace 能看到完整业务链路。
   - 当前状态：已完成。

## 阶段 3：真实 Agent 与工具能力，V0.12

目标：让 Agent 从“资产配置 + 模型调用”升级为“可组合、可治理、可扩展的执行单元”。

8. V0.12A：Agent Runtime 抽象。
   - 定义 Agent 输入、输出、错误、Token、成本和工具调用协议。
   - 验收：不同 Agent 节点按统一协议执行。
   - 当前状态：已完成。
9. V0.12B：Tool / Skill 资产库。
   - 独立管理工具、权限、参数 Schema 和调用日志。
   - 验收：Agent 只能调用已授权工具。
   - 当前状态：已完成。
10. V0.12C：MCP / HTTP 工具适配。
   - 接入 MCP Server 或企业 HTTP API。
   - 验收：工具调用结果进入节点产出物和 Trace。
   - 当前状态：HTTP Tool 测试调用、Agent 执行调用日志、Trace 事件、HTTP allowlist Gateway 和可注入 MCP 测试调用骨架已完成；真实 MCP Server client 待具体目标确定后接入。
11. V0.12D：LLM-as-a-Judge。
   - Rubric 可选择确定性评分器或 LLM Judge。
   - 验收：评分结果带理由、模型、版本和可复现输入。
   - 当前状态：可注入 Judge Gateway、Rubric judge 类型、Evaluation evaluator 元数据、ModelGateway JSON 解析、返回 schema 校验、解析失败重试、Prompt 版本记录、前端配置入口和校准概览已完成；更深一致性评估与成本统计待继续。

## 阶段 4：工作流调度与生产可靠性，V0.13

目标：让工作流从同步原型升级为可恢复、可观测、可治理的调度系统。

12. V0.13A：异步任务队列。
   - 引入后台执行队列，支持长任务不阻塞 API。
   - 候选工具：Celery / Dramatiq / RQ；后续生产可评估 Temporal。
   - 当前状态：第一版数据库队列、Worker、租约、心跳、退避、死信、取消和重投已实现；不等同生产消息队列。
13. V0.13B：工作流暂停、恢复、取消和重试。
   - 节点级重试、人工恢复、失败继续策略。
   - 验收：失败节点可定位、重试、恢复。
   - 当前状态：Human Review 恢复、失败节点恢复、Run 重跑和队列取消/重投第一版已实现。
14. V0.13C：条件路由、并行和汇聚。
   - 支持分支表达式、并行节点、汇聚策略。
   - 验收：复杂 DAG 能正确执行并保存节点结果。
   - 当前状态：尚未形成完整通用契约，不是当前 V1.0 Lite 签收阻断项。
15. V0.13D：实时事件推送。
   - SSE 或 WebSocket 推送运行状态。
   - 验收：运行中心无需刷新即可更新。
   - 当前状态：统一执行事件与 Trace 查询已实现；SSE/WebSocket 尚未实现。

## 阶段 5：企业化与 V1.0 发布

目标：满足真实组织使用的安全、部署、运维和试点要求。

16. V0.14A：组织、角色和权限硬化。
   - 完整 RBAC、成员审计、权限测试。
   - 验收：跨 Workspace 数据不可见，敏感操作有审计。
   - 当前状态：单组织、多 Workspace、固定 RBAC 和审计第一版已实现；企业 SSO 和多组织 SaaS 未实现。
17. V0.14B：外部通知。
   - 消费 NotificationOutbox，接入飞书通知。
   - 验收：SLA 到期、任务分派、运行失败可通知到人。
   - 当前状态：Outbox、Worker、渠道资产和失败治理已实现；真实外部渠道覆盖与业务验收不完整。
18. V0.14C：PostgreSQL 与迁移体系。
   - 正式迁移工具、初始化脚本、备份恢复。
   - 验收：SQLite 开发数据与 PostgreSQL 生产环境路径清晰。
   - 当前状态：PostgreSQL 连接和 Zeabur PostgreSQL 部署存在；正式迁移、备份恢复与演练未完成。
19. V0.14D：部署与环境管理。
   - Docker Compose、环境变量模板、启动/停止脚本。
   - 验收：新机器可按文档启动。
   - 当前状态：Docker、Compose、Windows 脚本和 Zeabur Runbook 已存在；本地 Python 环境当前不可重复。
20. V0.14E：CI/CD 与质量门禁。
   - 自动运行测试、lint、build、迁移检查。
   - 验收：未通过门禁不能合并。
   - 当前状态：GitHub CI 和 Zeabur exact-SHA 部署门禁已实现；数据库迁移门禁和自动回滚未完成。
21. V0.15A：试点业务流。
   - 选 1-2 条真实业务流程落地。
   - 验收：真实用户能从输入到产出再到评估闭环跑通。
   - 当前状态：默认种子资产与自动技术验收已完成；真实业务方独立签收未完成。
22. V1.0 RC：稳定性与性能压测。
   - 并发运行、失败恢复、日志与监控。
   - 验收：试点场景连续运行稳定。
23. V1.0：正式发布。
   - 冻结范围、交付部署包、验收手册和运维手册。
   - 验收：业务方按手册独立完成核心流程。

## 关键开源工具候选

- 前端：React、TypeScript、Vite、React Router、React Flow、Lucide。
- 服务端：FastAPI、Pydantic、SQLAlchemy。
- 数据库：SQLite 开发，PostgreSQL 生产。
- 测试：Vitest、Testing Library、Pytest、Playwright。
- 队列：Dramatiq / Celery / RQ，生产复杂编排可评估 Temporal。
- 观测：OpenTelemetry、Prometheus、Grafana、Loki。
- AI 接入：OpenAI-compatible API、MCP、后续可补 LiteLLM 或自研 ModelGateway 扩展。

## 当前推荐下一步

下一步不是新增 V0.x 功能，而是按顺序完成：

1. P0 运行时安全人工签收。
2. V1.0 Lite 真实业务方独立验收。
3. 根据试点证据选择可靠性治理或一个真实外部集成。

真实 MCP Server、更多通知渠道、LLM Judge 深度校准和复杂 DAG 都保留为候选项，只有
当真实试点明确需要并且安全/权限/失败/审计边界可验证时才立项。
