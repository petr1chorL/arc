# 领域文档使用规范

## 开始探索前

按需阅读：

- 根目录 `CONTEXT.md`。
- `.scratch/` 下当前功能的 PRD 和 Issue。
- 涉及当前能力时阅读 `docs/CURRENT_IMPLEMENTATION.md`。
- `docs/adr/` 存在时，阅读相关 ADR。

如果 ADR 目录或文件不存在，直接继续，不需要提前创建。领域文档和 ADR 只在
真正形成领域术语或关键决策时按需建立。

## 当前结构

ARC.ONE 当前使用单一领域上下文：

```text
/
├─ CONTEXT.md
├─ docs/
│  └─ adr/        # 首个符合条件的 ADR 出现时再创建
└─ src/
```

## 词汇要求

以下内容必须优先采用 `CONTEXT.md` 中的标准术语：

- PRD 与 Issue。
- 平台界面文案。
- 测试名称。
- 领域类型和接口。
- 架构讨论。

不得无说明地使用 `_避免使用_` 中列出的同义词。

## ADR 冲突

新方案与已有 ADR 冲突时，必须显式说明冲突原因，不能静默覆盖原决定。
