# V0.7B Review Workbench Design

## 目标

让人工审核页面成为可验收的工作台入口，而不是在无任务时给用户一个无法行动的空页面。

## 设计

- Reviews 页面继续使用真实 Human Task API，不引入演示数据接口。
- 页面顶部增加四个轻量运营指标：待处理任务、我的参与范围、SLA 风险、待确认反馈。
- 无任务状态解释 Human Task 的来源，并提供工作流编排与成员权限入口。
- 队列筛选支持全部 Human Task 状态；筛选无结果时提供清空筛选。

## 数据流

```text
listHumanTasks + listReviewers + listReviewGroups + listFeedbackCandidates
→ Reviews 页面本地聚合
→ 概览指标 / 空状态 / 队列筛选 / 详情工作台
```

## 异常处理

- API 加载失败沿用现有错误面板和重试按钮。
- 无任务不是错误，而是可行动的空状态。
- 无 Reviewer 资格时继续禁用审核动作，并在概览中提示需要配置资格。

## 测试

- 组件测试覆盖空状态入口。
- 组件测试覆盖概览与筛选无结果。
- 现有审核动作测试作为回归保护。
