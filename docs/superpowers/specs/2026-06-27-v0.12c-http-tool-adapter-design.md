# V0.12C HTTP 工具适配第一片设计

## 决策

第一片只实现可注入 HTTP 工具适配契约和测试调用接口。默认生产网关不主动访问外部网络，避免在没有安全策略、允许名单和凭证治理前引入不可控外联。

## 合约

- `ToolRuntimeExecutor.execute_http`：接收适配配置和参数。
- `ToolRuntimeResult`：包含状态、输入摘要、输出摘要、错误和耗时。
- `HttpToolGateway`：可注入网关，测试使用 FakeGateway；默认实现返回配置错误。
- `HttpxToolGateway`：默认 HTTP 实现，支持 GET / POST、host allowlist、超时和响应摘要。
- `McpToolGateway`：可注入 MCP 网关，当前用于测试调用和日志骨架；默认不连接真实 MCP Server。
- `ExecutionService`：Agent 执行时调用已绑定的 HTTP Tool，并写入带 Agent / Run / NodeRun 上下文的调用日志。
- `observability_execution_events`：把 Tool 调用日志派生成 `tool_skill_invocation` 事件，并关联 NodeRun Span。

## 数据

Tool / Skill 资产新增：

- `adapter_type`：`manual`、`http` 或 `mcp`。
- `adapter_config`：JSON 配置。HTTP 第一片只保存配置并交给注入网关解释。
