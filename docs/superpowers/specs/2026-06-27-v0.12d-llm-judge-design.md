# V0.12D LLM-as-a-Judge 第一切片设计

## 决策

第一切片只引入可注入 Judge Gateway 与持久化 evaluator 元数据。默认不主动调用模型，避免在 Prompt、JSON schema、重试和成本核算未定前扩大外部调用面。

## 合约

- `JudgeGateway`：接收 Rubric 快照、Rubric 版本、产出物文本和被评估对象上下文。
- `JudgeGatewayResult`：返回维度分、总分、状态、理由、模型和输入快照。
- `Rubric.judge_type`：`deterministic` 或 `llm`。
- `Evaluation.evaluator_*`：记录实际评分器类型、模型和可复现输入。
