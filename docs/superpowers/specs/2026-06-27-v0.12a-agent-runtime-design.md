# V0.12A Agent Runtime 抽象设计

## 决策

第一版 Runtime 是后端内聚模块，不新增数据库表，也不改变 ModelGateway 接口。Runtime 负责把 Agent Version Snapshot 转换后的执行请求交给 ModelGateway，并返回结构化结果；ExecutionService 继续负责读取版本、创建 NodeRunRecord 和保存运行。

## 合约

- `AgentRuntimeRequest`：描述一次 Agent 执行的业务上下文和模型输入。
- `AgentRuntimeResult`：描述执行状态、输出、脱敏错误、模型、Token、成本、评分、尝试次数、耗时和工具调用占位。
- `AgentRuntimeExecutor`：执行重试、成本计算、质量评分和错误脱敏。

## 取舍

- 不把 Runtime 结果直接持久化为 JSON 字段，避免现在引入迁移。
- 保留 `ModelGateway.complete(system_prompt, user_input, model)`，避免同时改网关和真实 DeepSeek 接入。
- `tool_calls` 先返回空列表，作为 V0.12B / V0.12C 的稳定挂点。

## 验证

- 新增 Runtime 单元测试。
- 复用现有 Agent 测试运行和工作流运行测试，证明两条入口仍走统一 `execute_agent` 路径。
