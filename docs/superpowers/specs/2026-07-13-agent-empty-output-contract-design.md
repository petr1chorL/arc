# Agent 空输出执行契约设计

## 决策

在 `AgentRuntimeExecutor` 的模型响应边界校验非空产出。空字符串或纯空白内容属于一次可重试的
无效模型响应；若节点允许的尝试全部无效，则返回脱敏失败结果。Workflow 复用现有失败节点中止、
Run 失败、异步任务重试/死信和失败节点恢复机制。

## 第一性原理

Agent 节点的输入与输出是两个不同事实。只有模型实际生成了非空输出，节点才具备“已完成”的
最低条件；将输入回退为输出会破坏运行证据、Artifact 追溯和 Human Review 的业务语义。

本切片只建立最低执行契约：

1. 模型调用成功不等于 Agent 执行成功。
2. 非空输出是成功的必要条件，但不是充分质量保证。
3. 空输出应先有限重试，耗尽后进入已有失败闭环。
4. 短但非空输出继续由现有质量分和人工复核处理。

## 方案比较

### 方案 A：继续把空输出计 0 分并创建人工复核

拒绝。人工审核没有 Artifact 可审核，并会把模型执行失败伪装成质量判断。

### 方案 B：在工作流层看到空输出后改为失败

拒绝。Agent 直接测试运行也会产生同样的假成功，而且无法在 Runtime 内利用现有尝试次数重试。

### 方案 C：在 Runtime 响应边界校验并有限重试（采用）

该位置同时覆盖 Agent 直接运行和工作流节点；变更不需要数据库迁移，也不改变短文本质量门禁。

## 状态与数据流

```text
ModelGateway 返回内容
  ├─ 非空：AgentRuntimeResult(已完成) -> 现有质量门禁
  └─ 空白：消耗一次 attempt
       ├─ 尚可重试：再次调用 ModelGateway
       └─ 已耗尽：AgentRuntimeResult(失败)
                    -> NodeRun 失败
                    -> 停止 DAG
                    -> Run 失败
```

失败结果的 `output_text` 为空、`score` 为 0，并使用固定中文错误摘要；不持久化 Provider 原始
错误或空响应正文。已成功调用所产生的 Token/成本是否可计量取决于 ModelGateway 返回数据，
本切片维持现有失败结果不计量的口径，不扩展计费审计。

## 对抗式边界

- 不以“输出等于输入”判失败，避免误伤确需复述输入的合法 Agent。
- 不以 0 分统一判失败；当前 0 分只由空白文本产生，但执行契约仍以空白校验为事实来源。
- 不校验 JSON、字段完整性或业务正确性；这些属于 Data Object/评估契约。
- 不新增跨 Workspace 查询、外部调用、密钥读取、日志或遥测。
- Human 节点只会在上游成功后创建，失败路径不得产生无内容审核任务。

## 验证设计

1. Runtime 单元测试：第一次空白、第二次有效，证明会重试并成功。
2. Runtime 单元测试：空白耗尽，证明返回固定脱敏失败且 attempts 正确。
3. API 端到端测试：空白耗尽后只存在 Start 与失败 Agent，Run 失败，无 Artifact/Review。
4. 既有短文本测试：继续得到 50 分并创建 Human Review。
5. 完成后运行后端全量、前端测试、lint、build、部署检查和浏览器验证。
