# ARC.ONE Soft UI 视觉重设计规格

> 日期：2026-06-24  
> 状态：待用户审阅  
> 范围：V0.1 前端原型的全站视觉系统与现有页面样式  
> 参考方向：Neomorphism、Soft UI、Soft Shadow Interface、Tactile Minimal UI

## 1. 设计目标

在不改变 ARC.ONE 当前功能结构和信息架构的前提下，把现有“深色企业控制台”
视觉升级为具有柔和材质感和操作触感的浅色 Soft UI。

设计需要同时满足：

1. 接近参考图的浅色同材质、柔和浮雕和内凹控件。
2. 保留企业平台所需的信息密度、扫描效率和状态识别。
3. 让工作流编排页成为最有辨识度的核心界面。
4. 不因追求 Neomorphism 而降低文字对比度和操作可发现性。
5. 不改变现有页面功能、路由、数据模型和业务交互。

## 2. 已确认的设计选择

### 2.1 Soft UI 强度

采用 **平衡触感界面**：

- 面板具有清晰但克制的外凸阴影。
- 输入框、搜索框、选中导航和按下状态采用内凹阴影。
- 不使用过深浮雕或大面积厚重阴影。
- 保持现有控制台的信息密度。

### 2.2 主色

采用 **雾蓝 × 珊瑚**：

- 雾蓝用于品牌、主操作、运行状态、选中态和主要图表。
- 珊瑚用于失败、风险、人工审核和少量高优先级提示。
- 页面背景和主要面板使用近白蓝灰同色体系。
- 不使用大面积渐变和高饱和紫色。

### 2.3 应用外壳

采用 **浅色悬浮导航**：

- 左侧导航与内容区使用相同材质。
- 导航栏作为独立外凸控制器悬浮在页面背景上。
- 当前导航项采用内凹状态，不使用深色整块背景。
- 移动端继续使用底部导航，并采用相同触感语言。

### 2.4 工作流节点

采用 **纯材质节点**：

- 正常节点统一使用雾蓝灰材质。
- 通过图标、标题和副标题识别节点类型。
- 不使用贯穿节点的类型色边。
- 人工审核、风险和失败状态可以使用珊瑚色图标或状态点。
- 选中节点使用低饱和蓝色轮廓，不改变节点主体材质。

## 3. 设计原则

### 3.1 材质统一

同一层级的组件使用相同背景色。组件层级主要通过光源方向、阴影深度和间距表达，
而不是依赖大量边框和背景色差。

统一假定光源来自左上方：

- 左上高光。
- 右下阴影。
- 内凹控件使用相反方向的内阴影。

### 3.2 触感服务于操作

外凸表示：

- 独立面板。
- 可点击按钮。
- 可拖动工作流节点。
- 悬浮导航。

内凹表示：

- 输入框。
- 搜索框。
- 当前选中导航。
- 当前激活的分段控件。
- 按钮按下状态。

阴影不能只作为装饰。没有层级或交互意义的元素不增加阴影。

### 3.3 信息密度优先

- 表格仍采用连续行和轻分隔线，不把每一行做成独立卡片。
- 页面区域不嵌套多层卡片。
- 统计指标可以共享一个外凸容器，内部使用留白和轻分隔。
- 工作流画布保持尽可能大的可操作面积。

### 3.4 状态不能只依赖颜色

状态同时使用：

- 图标。
- 状态文字。
- 状态点或轮廓。
- 必要时使用背景或阴影变化。

雾蓝和珊瑚不作为唯一识别信号。

## 4. 视觉 Token

以下值作为实施阶段的初始基线，可在浏览器验证时做小幅校准。

### 4.1 颜色

```css
:root {
  --soft-canvas: #e9edf5;
  --soft-surface: #e9edf5;
  --soft-highlight: #ffffff;
  --soft-shadow: #cbd0da;

  --soft-ink: #3d4558;
  --soft-muted: #818a9f;
  --soft-faint: #a2a9b7;
  --soft-line: rgba(101, 121, 168, 0.14);

  --soft-primary: #6579a8;
  --soft-primary-hover: #596d9b;
  --soft-primary-light: #94a4c8;
  --soft-primary-faint: #dce2ef;

  --soft-coral: #e68f6b;
  --soft-coral-dark: #c97555;
  --soft-coral-faint: #f3ddd4;

  --soft-success: #5f8f82;
  --soft-danger: #c96e68;
}
```

### 4.2 阴影

```css
:root {
  --shadow-raised-sm:
    4px 4px 9px rgba(184, 190, 202, 0.72),
    -4px -4px 9px rgba(255, 255, 255, 0.86);

  --shadow-raised-md:
    8px 8px 18px rgba(184, 190, 202, 0.72),
    -8px -8px 18px rgba(255, 255, 255, 0.86);

  --shadow-inset:
    inset 4px 4px 8px rgba(184, 190, 202, 0.72),
    inset -4px -4px 8px rgba(255, 255, 255, 0.88);

  --shadow-pressed:
    inset 3px 3px 7px rgba(177, 184, 198, 0.78),
    inset -3px -3px 7px rgba(255, 255, 255, 0.82);
}
```

阴影在移动端和小组件中按比例减弱，不直接复制桌面大阴影。

### 4.3 圆角

```css
:root {
  --radius-control: 9px;
  --radius-node: 12px;
  --radius-panel: 14px;
  --radius-shell: 16px;
}
```

主要圆角范围为 9-16px。除状态徽标、头像和小型开关外，不使用完全胶囊形状。

### 4.4 动效

- Hover：`160-200ms`，轻微抬升或阴影增强。
- Pressed：切换为内凹阴影，不缩放整个布局。
- 页面进入：只对一级内容使用一次轻微淡入和上移。
- 工作流运行节点：使用克制的轮廓呼吸，不使用大面积闪烁。
- 尊重 `prefers-reduced-motion`。

## 5. 应用外壳

### 5.1 页面背景

- 全局使用 `--soft-canvas`。
- 不使用装饰性渐变球、光斑或纹理图片。
- 可使用极低对比度的噪点，但第一版不引入额外位图资源。

### 5.2 左侧导航

- 桌面宽度约 `84-92px`，由当前文字导航改为图标优先的窄导航。
- Logo、导航图标和用户入口垂直排列。
- 鼠标悬停时通过 Tooltip 显示完整模块名称。
- 当前导航项为内凹方形控制面。
- 人工审核数量使用小型珊瑚角标。
- 工作空间名称移至用户入口 Tooltip 或展开面板，避免窄导航文字拥挤。

### 5.3 顶部区域

- 顶部栏不再使用整块白色横条和底边框。
- 页面标题直接位于内容背景上。
- 搜索框使用内凹材质。
- 通知按钮使用小型外凸圆角方形。
- 环境状态保留文字和状态点。

### 5.4 页面内容

- 桌面保持紧凑边距，主要区域间距 `16-20px`。
- 一级面板使用中等外凸阴影。
- 内容高度较长时，背景保持连续，不将页面本身包装为一张大卡片。

## 6. 通用组件

### 6.1 按钮

**主要按钮**

- 雾蓝实色。
- 文字使用白色。
- 默认有小型外凸阴影。
- 按下时阴影转为内凹。

**次要按钮**

- 与页面同色。
- 外凸小阴影。
- Hover 时文字和图标转为主色。

**危险按钮**

- 不使用大面积红色填充。
- 使用珊瑚或危险红文字、图标和浅色轮廓。

### 6.2 输入控件

- 输入框、选择框、搜索框使用内凹阴影。
- Focus 时增加雾蓝轮廓和清晰的键盘焦点。
- Placeholder 对比度不得过低。
- 错误状态使用珊瑚文字、图标和错误说明。

### 6.3 面板

- 统一使用同色背景和外凸阴影。
- 标题区与内容区优先使用留白，不使用粗边框。
- 必须分隔时采用低对比度细线。
- 面板内部不再嵌套具有同等阴影的卡片。

### 6.4 状态徽标

- 保持小尺寸。
- 使用浅色同系背景、状态点和文字。
- 不使用重阴影。
- 高风险和人工审核使用珊瑚色系。

### 6.5 表格

- 表格整体位于一个外凸面板中。
- 表头使用轻微内凹或低对比度底色，不使用独立卡片。
- 数据行使用细分隔线和 Hover 浅色变化。
- 选中行可使用轻微内凹背景。
- 保持列对齐和当前信息密度。

## 7. 页面设计

### 7.1 运营总览

- 今日态势区域使用横向外凸面板。
- 指标区保留共享容器，六项指标之间使用轻分隔。
- 趋势图和关注事项各为一级外凸面板。
- 图表使用雾蓝深浅，不增加多色系列。
- 失败、人工审核和即将超时使用珊瑚。

### 7.2 Agent 资产

- 顶部概览使用共享外凸容器。
- Agent 列表继续采用表格，不改为卡片瀑布流。
- Agent 图标使用小型内凹底座。
- 搜索框和筛选控件使用内凹与外凸组合。

### 7.3 评估中心

- 评估概览使用共享面板。
- Rubric 可以保留单项面板，但降低阴影强度，避免页面过于起伏。
- 权重轨道使用内凹槽，权重值使用雾蓝填充。
- 硬性门禁使用低饱和雾蓝底色；风险门禁才使用珊瑚。

### 7.4 运行中心

- 左侧运行列表位于一个外凸面板。
- 当前运行项使用内凹选中态。
- 右侧运行详情为一级外凸面板。
- 时间线节点使用内凹小圆点或浅浮雕标记。

### 7.5 人工审核

- 审核队列和详情各为一级外凸面板。
- 当前审核任务使用内凹选中态。
- 人工判断原因和扣分项使用珊瑚，但控制面积。
- 底部审核操作区保持固定并使用柔和顶部分隔。

## 8. 工作流编排页

### 8.1 画布

- 使用雾蓝灰背景和低对比度点阵。
- 画布本身不加厚重外凸阴影，以免压缩视觉空间。
- 顶部工具栏、左侧节点库和右侧配置栏分别作为悬浮外凸控制器。

### 8.2 工作流节点

- 正常节点统一使用同色外凸材质。
- 图标置于小型内凹底座。
- 选中节点增加 `2px` 低饱和雾蓝轮廓。
- 正在运行节点使用雾蓝轮廓呼吸。
- 人工审核、失败和风险节点使用珊瑚色图标或状态点。
- 不使用节点类型色边，不改变主体背景色。

### 8.3 连线

- 默认使用中性蓝灰。
- 运行中的连线使用主雾蓝并保留动画。
- 失败或阻断路径可以使用珊瑚。
- 连线标签保持高对比度，不能被画布点阵吞没。

### 8.4 配置面板

- 配置面板为悬浮外凸区域。
- 表单控件全部使用内凹样式。
- 开关保持清晰的开关位置和颜色，不只依赖阴影。

## 9. 移动端

- 底部导航使用浅色悬浮控制条。
- 当前导航项使用内凹状态。
- 页面内容不强制保留桌面多栏布局。
- 审核和运行页面保持纵向堆叠。
- 工作流编排隐藏节点库，配置面板以侧滑层呈现。
- 移动端阴影半径和偏移减少约 35%，避免界面发糊。
- 所有文本不得因 Soft UI 圆角或阴影发生裁切。

## 10. 可访问性

- 正文文字和背景对比度满足 WCAG AA。
- 重要操作和状态不只依赖阴影。
- 键盘焦点使用明确的雾蓝轮廓。
- 图标按钮必须保留 Tooltip 或 `aria-label`。
- 支持 `prefers-reduced-motion`。
- 支持 `prefers-contrast: more` 时减弱阴影并增加边框。

## 11. 技术实现边界

### 保持不变

- React、TypeScript、Vite。
- React Router 路由结构。
- React Flow 节点和连线逻辑。
- 当前模拟数据。
- 页面功能和交互。

### 允许调整

- `Layout.tsx` 的导航和顶部区域结构。
- 通用组件的样式类和少量语义结构。
- 页面布局的间距与容器结构。
- `WorkflowNode.tsx` 的节点视觉结构。
- `index.css` 的 Token 和组件样式。

### 不在本次范围

- 后端和数据库。
- 新业务功能。
- 页面数据结构。
- 模型调用。
- 全新 UI 组件框架。
- 暗色模式。
- 自定义图片或插画。

### 12.3 验收标准

- [ ] 全站使用统一的雾蓝灰 Soft UI 材质。
- [ ] 左侧导航为浅色悬浮控制器。
- [ ] 面板外凸、输入内凹、按下内凹的规则一致。
- [ ] 珊瑚色只用于风险、失败、人工审核和重点提示。
- [ ] 工作流正常节点保持纯材质，不使用类型色边。
- [ ] 表格仍然易扫描，没有卡片套卡片。

- [ ] 移动端不存在横向页面溢出。

- [ ] 现有交互行为不回退。
- [ ] 浏览器控制台没有新增错误。

## 13. 设计结果摘要

最终视觉方向是：

> 一台浅色雾蓝灰的企业 AI 控制台。界面通过克制的外凸和内凹光影提供触觉感，
> 用雾蓝表达主操作与运行状态，用珊瑚表达人工介入和风险。工作流节点保持统一
> 材质，让复杂流程依靠结构、图标和状态而非多色分类被理解。
