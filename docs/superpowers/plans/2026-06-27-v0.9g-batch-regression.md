# V0.9G Batch Regression Implementation Plan

## Goal

在 Evaluation Center 增加一个轻量批量回归入口：运营或评估负责人选择一个 Rubric，输入多条样本后连续运行已有单条评估 API，并立即看到通过率、失败样本和落入历史记录的评估结果。

## Scope

- 复用现有 `evaluateRubric` API，不新增后端任务表。
- 每条样本都作为一条 `EvaluationRecord` 持久化。
- 页面显示批量摘要、失败样本、每条样本得分和状态。
- 支持前端必填校验与运行中状态。

## Out of Scope

- 不做定时回归任务。
- 不做后台队列、重试、并发控制和任务历史表。
- 不做跨工作流的自动样本集管理。

## Architecture

前端批量回归面板解析输入样本，按顺序调用现有 Rubric 评估接口。接口返回的记录直接插入 Evaluation History，保证用户能在同一个页面继续查看详情。

## Tech Stack

- React + TypeScript
- Vitest + Testing Library
- 现有 REST API: `/api/workspaces/:workspaceId/evaluations/rubrics/:rubricId/evaluate`
- 现有 Soft UI 样式系统
