# Workflow Data Object Binding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let workflow builders bind Data Object Definitions to node input and output contracts.

**Architecture:** Reuse the existing workflow node `data` object as the persistence surface. The Workflows page loads Data Object Definitions with workspace directories, passes them into `NodeInspector`, and saves selected refs through the existing `toContractGraph` workflow draft path.

**Tech Stack:** React, TypeScript, React Testing Library, Vitest, existing FastAPI-backed workspace APIs.
