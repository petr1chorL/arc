# V0.21D 运行中心选择同步 URL 实施计划

## 目标

让运行中心当前选中的 Run 可以通过 URL 恢复，补齐深链的“进入、切换、分享”闭环。

## 文件范围

- `src/pages/Runs.tsx`
- `src/pages/Runs.test.tsx`
- `docs/ACCEPTANCE_V0.21D.md`
- `docs/CURRENT_IMPLEMENTATION.md`

## 实施步骤

1. RED：新增 Runs 页面测试，点击第二条 Run 后断言 URL 包含该 Run 的 `runId`。
2. GREEN：新增 `syncRunIdToLocation`，手动选择 Run 时更新 URL。
3. GREEN：重跑、批量重跑、恢复等会自动选中 Run 的操作也同步 URL。
4. 文档：补 V0.21D 验收和当前实现说明。
5. 验证：focused 测试、完整测试、lint、build、浏览器验收。

## 验收标准

- 选中 Run 后 URL 变为包含 `runId` 的可分享地址。
- 已有 `?runId=...` 初始选中能力保持可用。
- 运行操作后被自动选中的 Run 也会同步到 URL。
