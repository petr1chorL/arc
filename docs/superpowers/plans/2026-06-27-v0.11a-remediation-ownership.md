# V0.11A 实施计划：修复任务负责人、截止时间和筛选

## 范围

把 Remediation Task 从“可追踪修复项”推进到“可运营任务”：增加负责人、截止时间、逾期判断和列表筛选。

## 步骤

1. 后端红测：创建任务时保存 `owner`、`dueDate`，列表按 `owner`、`priority`、`overdue` 过滤。
2. 后端实现：模型字段、SQLite 增量补列、Pydantic 合同、列表查询参数和过滤逻辑。
3. 后端 focused 测试转绿。
4. 前端红测：任务卡展示负责人/截止时间/逾期状态，并支持筛选。
5. 前端实现：API 查询参数、类型字段、筛选控件、任务卡信息和空状态。
6. 前端 focused 测试转绿。
7. 全量验证：后端测试、前端测试、lint、build、浏览器验收。
8. 更新文档、验收记录和路线图，提交 V0.11A。

## 风险与约束

- 不新增完整任务评论系统，避免 V0.11A 范围膨胀。
- 不接入外部通知；逾期只在页面和 API 过滤中体现。
- 历史任务的 `owner` 和 `dueDate` 允许为空，但新创建任务要有默认值。
