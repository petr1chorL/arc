# V0.9D Rubric 评估运行实施计划

## 背景

V0.9C 已经支持 Rubric 创建、编辑、发布不可变版本和停用，但 Rubric 仍只是评分标准资产，还不能对产出物形成正式 Evaluation 记录。

V0.9D 补齐最小评估运行闭环：选择一个 active Rubric，对一段产出物执行评估，保存可追溯的 Evaluation 记录。

## 范围

- 新增 `evaluations` 数据表。
- 新增 `POST /evaluations/rubrics/{rubric_id}/evaluate`。
- 新增 `GET /evaluations/records`。
- 评估记录保存 Rubric 快照、Rubric 版本、产出物文本、维度得分、总分、状态和说明。
- 前端 Rubric 配置弹窗增加“运行评估”区。
- 运行评估后显示总分、状态、说明和维度分。

## 重要设计

- 本版不接入 LLM-as-a-Judge。
- 本版使用确定性评分器，分数基于产出物长度和质量信号词，用于验证数据链路。
- 优先使用最新已发布 RubricVersion 快照；如果没有发布版本，则使用当前 active Rubric 快照。
- draft 和 disabled Rubric 不允许运行评估。

## 不做

- 不做批量评估任务。
- 不做 Golden Set 回归。
- 不做 Langfuse / DeepEval / Ragas 集成。
- 不做真实模型评分。

## 验收标准

- 发布后的 Rubric 可以运行评估并返回 201。
- Evaluation 记录包含 `rubricSnapshot`，后续 Rubric 改动不影响历史记录。
- Evaluation 记录可以通过列表 API 查询。
- draft / disabled Rubric 运行评估返回 409。
- 前端配置弹窗能输入产出物、点击“运行评估”、看到总分和 passed/failed 状态。
