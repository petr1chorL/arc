# V0.21E 审计筛选同步 URL 实施计划

## 目标

让 Workspace 审计日志页面的筛选条件具备可恢复、可分享的 URL 状态，使 Run -> Audit 的 Trace 深链可以继续叠加动作、对象类型和结果筛选。

## 文件范围

- `src/pages/AuditLog.tsx`
- `src/pages/AuditLog.test.tsx`
- `docs/ACCEPTANCE_V0.21E.md`
- `docs/CURRENT_IMPLEMENTATION.md`

## 实施步骤

1. RED：在 AuditLog 页面测试中加入 location probe，断言修改筛选后 URL 会同步更新并保留既有筛选。
2. RED：确认当前实现只读取初始 `traceId`，修改筛选不会写回 URL。
3. GREEN：让 `traceId`、`action`、`targetType`、`outcome` 都从 URL 初始化。
4. GREEN：使用 `setSearchParams(..., { replace: true })` 在筛选变化时同步 URL，清空值时移除参数。
5. 文档：补充 V0.21E 验收文档和当前实现说明。
6. 验证：聚焦测试、回归测试、lint、build、浏览器验收。

## 验收标准

- 审计日志页面打开深链后能恢复 `traceId`、`action`、`targetType`、`outcome`。
- 用户修改筛选后，URL 无刷新同步更新。
- URL 中已有的无关查询参数不被删除。
- 审计 API 请求与当前筛选状态一致。
- 不新增后端接口，不改变审计权限模型。
