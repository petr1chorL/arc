# V0.16C Agent Tool Skill Binding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let Agent builders bind Tools and Skills from the Workspace asset library instead of relying only on free text.

**Architecture:** Reuse the existing `listToolSkillAssets` frontend API wrapper. Keep the existing Agent API payload shape (`tools: string[]`, `skills: string[]`) and add an asset-picker UI that updates the existing text-backed draft state by asset name.

**Tech Stack:** React, TypeScript, Vitest, Testing Library, existing FastAPI contracts.
