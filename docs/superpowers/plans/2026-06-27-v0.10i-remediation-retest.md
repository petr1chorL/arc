# V0.10I Remediation Retest Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let completed remediation tasks trigger a focused Regression Run retest and display the retest evidence on the task card.

**Architecture:** Store a nullable `retest_run_id` on `remediation_tasks`. Add a backend endpoint that builds a new Regression Run from the source run's failed sample records, then returns the updated remediation task with a compact `retestRun` summary. The frontend calls this endpoint from the task board and renders the result.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic, SQLite migration helper, React, TypeScript, Vitest, Pytest.
