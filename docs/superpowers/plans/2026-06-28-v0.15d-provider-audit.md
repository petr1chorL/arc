# V0.15D Provider Audit View Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let operators inspect recent audit events and rollback helper metadata directly from each model Provider card.

**Architecture:** Add a Provider-scoped audit endpoint that filters existing `AuditEventRecord` rows by Provider context from audit payload/metadata. The frontend loads audit events alongside Provider impact and renders a compact “最近变更” section per Provider card, including migration rollback helper details without implementing rollback execution.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic schemas, React, TypeScript, Vitest, Testing Library.
