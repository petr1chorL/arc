# V0.26F Artifact Schema API 校验状态实施计划

## 范围

给 Artifact 目录 API 增加后端派生的轻量 Schema 校验状态，并让前端优先消费该字段。

## 文件

- `apps/api/tests/test_execution_api.py`
- `apps/api/app/schemas.py`
- `apps/api/app/main.py`
- `apps/api/app/artifact_schema_validation.py`
- `src/types.ts`
- `src/api/artifacts.test.ts`
- `src/pages/Artifacts.tsx`
- `docs/CURRENT_IMPLEMENTATION.md`
- `docs/ACCEPTANCE_V0.26F.md`

## 步骤

1. 在后端 API 测试中增加 RED：合法 Artifact 返回 `passed`，缺少 required 字段返回 `failed` 和原因。
2. 新增后端轻量校验 helper，并把 `schemaValidation` 加入响应模型和 `/artifacts` 装配。
3. 跑后端聚焦测试确认 GREEN。
4. 更新前端类型和 API 测试，页面优先读取 API 字段，旧响应 fallback 本地校验。
5. 跑前端 Artifact 相关测试。
6. 更新当前实现说明和 V0.26F 验收文档。
7. 跑 lint、build 和浏览器验证后提交。
