# V0.9E 评估记录中心实施计划

## 背景

V0.9D 已经能运行 Rubric 评估，并把 Evaluation 记录保存到后端，但用户还不能在评估中心直接查看历史记录，只能通过接口或运行后的弹窗结果确认。

V0.9E 补齐最小历史工作台：把 Evaluation 记录展示出来，并支持基础筛选，方便后续做评估集、回归任务和质量趋势。

## 范围

- 评估中心加载 Evaluation 记录列表。
- 页面新增“评估记录”区块。
- 每条记录展示 Rubric 快照、评估对象、版本、维度分、总分、状态和说明。
- 支持按 `passed` / `failed` 状态筛选。
- 支持按 Rubric 筛选。
- 运行评估成功后，将新 Evaluation 记录插入历史列表。

## 重要设计

- 历史记录展示使用 `rubricSnapshot`，避免 Rubric 后续编辑影响历史解释。
- Rubric 筛选项优先使用当前 Rubric 列表，同时补充历史记录里已不在当前列表中的 Rubric 快照名称。
- 筛选为空时显示空状态，不隐藏评估记录区块。
- 运行评估后不重新拉全量列表，先本地插入，减少等待；刷新页面后仍以 API 返回为准。

## 不做

- 不做评估记录详情页。
- 不做批量评估。
- 不做趋势图和质量报表。
- 不做 Golden Set 回归任务。

## 验收标准

- 评估中心能展示后端返回的 Evaluation 记录。
- 记录展示 ID、Rubric 快照名称、版本、对象、总分、状态、维度分和评分说明。
- 状态筛选能隐藏不匹配记录。
- Rubric 筛选能隐藏不匹配记录。
- 在 Rubric 弹窗运行评估后，新记录立即出现在历史列表中。
- 自动化测试覆盖记录加载、状态筛选、Rubric 筛选和运行后插入。
