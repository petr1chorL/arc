# V0.9A 评估资产概览实施计划

## 背景

评估中心仍主要展示 mock Rubric。人工审核闭环已经能产生 FeedbackCandidate，并由专家确认 GoldenSample，但评估中心没有展示这些真实评估资产。

## 目标

用最小端到端切片打通：

```text
人工修改
→ FeedbackCandidate
→ 专家确认 GoldenSample
→ 评估中心真实资产概览
```

## 非目标

- 不实现 Rubric CRUD。
- 不实现 LLM-as-a-Judge。
- 不实现回归测试执行器。
- 不接入 Langfuse、promptfoo 或 Ragas。

## 实施步骤

1. 后端新增 `GET /api/workspaces/{workspace_id}/evaluations/overview`。
2. API 汇总反馈候选、待确认候选、已确认候选、Golden Sample、覆盖工作流和覆盖 Agent。
3. API 返回最近 5 条反馈候选摘要。
4. 前端新增 `src/api/evaluations.ts`。
5. 评估中心加载真实概览，同时保留现有 Rubric 展示。
6. 增加后端和前端测试。
7. 完成 lint、build、全量测试和浏览器验收。

## 验收标准

- 评估中心能看到“评估资产概览”。
- 页面指标来自真实 API，而不是 mock。
- 页面展示反馈候选、Golden Sample、覆盖工作流、待确认候选。
- 没有反馈候选时给出明确空状态。
- 最近候选展示状态、原因和标签。
- 浏览器控制台无 error/warn。
