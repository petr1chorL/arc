# V0.24B Edge Field Mapping Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add editable workflow edge field mappings that persist in drafts and freeze into workflow versions.

**Architecture:** Store mapping rows inside the existing workflow edge contract under `edge.data.mappings`, avoiding a new endpoint or database table. Backend Pydantic schemas accept optional edge data, domain graph conversion preserves React Flow `Edge.data`, and the Workflows edge inspector edits mapping rows before the existing save draft request.

**Tech Stack:** FastAPI, Pydantic, SQLAlchemy JSON columns, React 19, TypeScript, React Flow, Vitest, Testing Library, Pytest.
