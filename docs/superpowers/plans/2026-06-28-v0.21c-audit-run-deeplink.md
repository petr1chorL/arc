# V0.21C 审计日志反向跳回运行中心实施计划

## 目标

把 V0.21B 的 Run -> Audit 链路补成双向追踪：审计日志中的 run 事件可以跳回运行中心，并自动选中对应 Run。

## 文件范围

- `src/pages/AuditLog.tsx`
- `src/pages/AuditLog.test.tsx`
- `src/pages/Runs.tsx`
- `src/pages/Runs.test.tsx`
- `src/index.css`
- `docs/ACCEPTANCE_V0.21C.md`
- `docs/CURRENT_IMPLEMENTATION.md`

## 实施步骤

1. RED：AuditLog 测试断言 run 审计事件存在“查看运行”链接。
2. RED：Runs 测试断言 `?runId=...` 会选中对应 Run。
3. GREEN：AuditLog 为 `targetType === "run"` 且存在 `targetId` 的事件渲染链接。
4. GREEN：Runs 初始选中和加载回退逻辑读取 URL 中的 `runId`。
5. 文档：补验收文档与当前实现说明。
6. 验证：focused 测试、完整测试、lint、build、浏览器验收。

## 验收标准

- 审计日志 run 事件跳转到 `/w/{workspaceSlug}/runs?runId={runId}`。
- 运行中心深链能选中对应 Run。
- 原有 Runs 操作历史、重跑、批量操作测试不回归。
