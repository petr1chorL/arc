# V0.23F 工作流撤销与重做实施计划

## 范围

为工作流编排页增加页面内撤销和重做能力，覆盖新增节点、复制节点、删除节点、删除连线和新增连线。

## 步骤

1. RED：新增“新增节点后撤销/重做”的 Workflows 测试，并确认当前缺少按钮而失败。
2. GREEN：增加画布历史快照、撤销/重做按钮和新增节点历史记录。
3. RED/GREEN：覆盖删除已连接节点，撤销恢复节点和关联连线，重做再次删除。
4. RED/GREEN：覆盖删除连线，撤销恢复连线，重做再次删除。
5. 重置：加载、新建、切换和保存成功后清空历史栈。
6. 验证：运行 Workflows 聚焦测试。
7. 回归：运行前端拆分测试、后端 pytest、lint 和 build。
8. 浏览器验收：登录本地 4173，新增节点后撤销/重做，删除节点或连线后撤销/重做。
9. 文档：更新 V0.23F 验收文档、当前实现说明和本地 Issue 状态。
