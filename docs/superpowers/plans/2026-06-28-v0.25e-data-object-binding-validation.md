# Data Object Binding Validation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prevent workflow publication when node Data Object bindings point to missing or unpublished definitions.

**Architecture:** Extend the existing `validate_workflow` domain function. The API already calls this validator from `/validate` and `/publish`; passing the current workspace ID lets validation query `DataObjectDefinitionRecord` without adding new endpoints or schema fields.

**Tech Stack:** FastAPI, SQLAlchemy, Pytest, existing Workflow validation API.
