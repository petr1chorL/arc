# V0.14D Runtime Provider Config Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Pass published Agent runtime configuration into the model gateway when an Agent test run or workflow Agent node executes.

**Architecture:** Extend `AgentRuntimeRequest` with non-secret runtime fields from the Agent version snapshot: provider ID, provider type, Base URL, temperature, and max output tokens. `ExecutionService` maps immutable snapshot fields into the runtime request. `ModelGateway.complete` accepts these optional runtime overrides; `OpenAICompatibleGateway` uses Base URL, temperature, and max tokens without exposing secrets.

**Tech Stack:** FastAPI service layer, Python dataclasses, ModelGateway protocol, pytest.

---

- `apps/api/tests/test_execution_api.py`: RED/GREEN end-to-end test that an Agent test run passes published runtime config to the gateway.

- `apps/api/app/model_gateway.py`: accept runtime override parameters and use them for OpenAI-compatible payloads.
