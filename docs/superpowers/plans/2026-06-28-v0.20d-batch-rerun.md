# V0.20D Batch Rerun Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let operators select multiple rerunnable Workflow Runs in the Runs page and rerun them in one action using each source Run's original input.

**Architecture:** Add a batch API endpoint that loops over requested source Run IDs, creates a new Workflow Run for each valid item, and records per-item failures instead of aborting the whole batch. The frontend adds a small selection layer to Runs, calls the batch endpoint, prepends created Runs, and selects the first created Run.

**Tech Stack:** FastAPI, Pydantic v2, SQLAlchemy, React, TypeScript, Testing Library, Vitest.
