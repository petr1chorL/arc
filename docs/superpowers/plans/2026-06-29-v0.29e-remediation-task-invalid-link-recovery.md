# V0.29E Remediation Task Invalid Link Recovery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let users recover from invalid Remediation Task deep links without manually editing the URL.

**Architecture:** Reuse the existing Evaluations page URL state. When `taskId` cannot resolve to a highlighted Remediation Task, render a “清除定位” action that removes only `taskId` from `useSearchParams`; the Remediation Tasks list remains loaded.

**Tech Stack:** React, TypeScript, React Router, Vitest, Testing Library, Playwright browser smoke check.
