# V0.10A Regression Runs Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Persist each batch regression as a first-class Regression Run with summary metrics and linked Evaluation records.

**Architecture:** Add a `regression_runs` workspace table that stores one run summary and a list of generated evaluation record IDs. Add a server-side batch endpoint that evaluates all selected samples using the existing deterministic Rubric evaluator, persists both the run and per-sample Evaluation records, and returns a detail payload for the UI. The Evaluations page reads recent runs and uses the new endpoint instead of client-side looping.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic, SQLite, React, TypeScript, Vitest, Testing Library.
