# V0.14B Provider Assets Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Workspace-scoped model Provider asset entry point with secret references and a safe connectivity test.

**Architecture:** Keep Provider configuration separate from Agent drafts. Persist only non-secret provider metadata and a secret reference label; never accept or return raw API keys. Add a backend connectivity test endpoint that validates configuration shape and reports whether the secret reference is resolvable without echoing secret values.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic, SQLite migration helper, React, TypeScript, Vitest, Testing Library, native CSS.
