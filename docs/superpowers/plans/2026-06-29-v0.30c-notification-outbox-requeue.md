# V0.30C Notification Outbox Requeue Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Allow failed Notification Outbox rows to be manually requeued with preserved failure evidence and audit context.

**Architecture:** Extend the V0.30A dispatcher service with a small `requeue_failed(...)` operation. Expose a Workspace API route that requires `workspace.manage`, only transitions `failed -> pending`, preserves the previous dispatch payload in history, and records an audit event.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic, Pytest, existing ARC.ONE audit/RBAC services.

---

## First-Principles Gate

- Bottom-line objective: notifications must have a recoverable path after channel failure.
- Minimal object/state needed: one failed outbox row, one transition back to pending, one reasoned audit event, preserved previous failure context.
- This is more fundamental than automatic retry because auto retry needs channel limits, idempotency keys, and external provider semantics that do not exist yet.

## Adversarial Review Gate

- Do not allow `sent` notification requeue; that can duplicate real messages later.
- Do not erase previous dispatch failure; preserve it in `dispatchHistory`.
- Do not claim this is production notification retry; it is manual requeue only.
- Do not bypass Workspace isolation; cross-Workspace records must return 404.
- Do not skip audit; a repeated send path needs a reason trail.

## Task 2: Implement Requeue

**Files:**
- Modify: `apps/api/app/notification_dispatcher.py`
- Modify: `apps/api/app/main.py`
- Modify: `apps/api/app/schemas.py`

- [x] **Step 1: Add service method**

Implement `NotificationOutboxDispatchService.requeue_failed(session, workspace_id, notification_id, reason)`:

- find current Workspace notification by id.
- return `None` if missing.
- raise conflict if status is not `failed`.
- append old `payload.dispatch` to `payload.dispatchHistory`.
- set status to `pending`.
- set `payload.dispatch` to pending metadata.

- [x] **Step 2: Add API route**

Add:

```text
POST /api/workspaces/{workspace_id}/notifications/outbox/{notification_id}/requeue
```

Require `workspace.manage`. On success, write `AuditEventRecord` with action `notification_outbox.requeue`, before status, after status, reason, and notification metadata.
