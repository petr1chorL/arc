# V0.20F Batch Operation Result Panel Implementation Plan

## Goal

Make batch run operations explain partial failures in the Runs page so an operator can identify which source Run failed and why.

## Files

- `src/pages/Runs.tsx`
- `src/pages/Runs.test.tsx`
- `src/index.css`
- `docs/ACCEPTANCE_V0.20F.md`
- `docs/CURRENT_IMPLEMENTATION.md`
- `.scratch/v0.20f-batch-operation-result/PRD.md`
- `.scratch/v0.20f-batch-operation-result/issue.md`

## Plan

1. Add RED tests for partial batch rerun and partial batch resume.
2. Store batch operation failures in Runs page state.
3. Render a compact failure result panel under the operation notice.
4. Clear stale failure results before refresh and new run operations.
5. Add soft UI styling for the failure result panel.
6. Run focused tests, full checks, and browser validation.
7. Update implementation and acceptance documentation.

## Acceptance

- Partial batch rerun exposes `sourceRunId` and `reason`.
- Partial batch resume exposes `sourceRunId` and `reason`.
- Existing successful batch operation behavior remains unchanged.
- Documentation records scope and validation evidence.
