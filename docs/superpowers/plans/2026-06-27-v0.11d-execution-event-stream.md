# V0.11D 执行事件流实施计划

## 目标

在运行观测详情中展示统一执行事件流，让一个 Trace 下的运行、节点、人工任务和审计事件按时间顺序可读；同时提供 Workspace 级事件查询，把修复任务、复测 Run 和复测失败活动纳入统一事件模型。

## 文件

- `apps/api/app/schemas.py`：新增执行事件 Read Schema。
- `apps/api/app/main.py`：构建并返回 `executionEvents`。
- `apps/api/tests/test_observability_api.py`：新增后端红测。
- `src/types.ts`：新增前端类型。
- `src/api/observability.ts`：新增 Workspace 级事件查询客户端。
- `src/api/observability.test.ts`：覆盖事件查询参数。
- `src/pages/Observability.tsx`：展示执行事件流。
- `src/pages/Observability.test.tsx`：新增 UI 红测。
- `src/index.css`：补事件流样式。
- `docs/ACCEPTANCE_V0.11D.md`、`docs/CURRENT_IMPLEMENTATION.md`、`docs/PROJECT_ROADMAP_TO_V1.md`：更新验收和版本说明。

## 步骤

1. 后端 RED：断言运行详情返回 `executionEvents`，包含 run、node、human_task、audit_event。
2. 后端 GREEN：新增 Schema 和聚合函数，按时间排序返回事件。
3. 后端扩展：新增 Workspace 级 `/execution-events` 查询，并覆盖修复任务、复测 Run 和活动记录。
4. 前端 RED：断言运行观测详情显示“执行事件流”和关键事件。
5. 前端 GREEN：新增时间线 UI、样式和事件查询客户端。
6. 验证：运行 focused 测试、全量测试、lint、build、浏览器验收。
