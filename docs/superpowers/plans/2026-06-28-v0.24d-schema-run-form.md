# V0.24D Schema Run Form Implementation Plan

**Goal:** Generate a lightweight workflow run input form from simple workflow `inputSchema` objects, then submit the generated JSON through the existing run API.

**Architecture:** Keep the backend contract unchanged. The Workflows page derives a list of one-level schema fields from `currentWorkflow.inputSchema`. If fields are supported, the run dialog renders inputs and serializes values into `runWorkflow(..., { input: JSON.stringify(payload) })`. If no supported fields exist, it keeps the existing textarea behavior.
