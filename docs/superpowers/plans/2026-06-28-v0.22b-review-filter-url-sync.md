# V0.22B 人工审核筛选同步 URL 实施计划

## 目标

在 V0.22A 的 `taskId` 深链基础上，把人工审核页的状态筛选和 SLA 筛选也同步到 URL，补齐审核队列视图的分享和刷新恢复能力。

## 文件范围

- `src/pages/Reviews.tsx`
- `src/pages/Reviews.test.tsx`
- `docs/ACCEPTANCE_V0.22B.md`
- `docs/CURRENT_IMPLEMENTATION.md`

## 实施步骤

1. RED：新增 Reviews 测试，断言 URL 中的 `taskStatus` 和 `slaStatus` 会初始化筛选控件。
2. RED：同一测试断言修改筛选后 URL 保留 `taskId` 并同步筛选参数。
3. GREEN：从 `searchParams` 初始化 `statusFilter` 和 `slaFilter`。
4. GREEN：在 `selectedId`、`statusFilter`、`slaFilter` 变化时统一同步 URL。
5. GREEN：`全部` 时删除 `taskStatus` 或 `slaStatus`。
6. 文档：补充验收文档和当前实现说明。
7. 验证：聚焦测试、全量回归、lint、build、浏览器验收。

## 验收标准

- URL 可恢复 `taskId + taskStatus + slaStatus`。
- 用户修改筛选后，URL 无刷新同步更新。
- 筛选清空时删除对应查询参数。
- 不改变人工审核动作和权限契约。
