# V0.21A Run 操作历史实施计划

## 目标

让用户在运行中心选中任意 Run 后，可以看到该 Run 关联的运行操作历史，包括单条重跑、批量重跑、失败点恢复和批量恢复。

## 步骤

1. 后端 RED：增加 Run 操作历史接口测试，确认接口尚未存在时 404。
2. 后端 GREEN：增加响应 schema、关联判断函数和只读接口，从审计事件中过滤目标 Run 相关操作。
3. 前端 API RED：增加 `listRunOperationHistory` 请求路径测试，确认方法缺失时失败。
4. 前端 API GREEN：实现 `listRunOperationHistory` 与 `RunOperationHistoryEvent` 类型。
5. 页面 RED：增加 Runs 页面操作历史渲染测试，确认页面缺少区块时失败。
6. 页面 GREEN：Runs 详情页加载并展示操作历史、加载态、空态和错误态。
7. 样式：保持 soft UI 卡片风格，避免与 KPI、最终产出和时间线拥挤。
8. 文档：更新当前实现说明与验收文档。
9. 验证：运行 focused 测试、完整前端测试、后端测试、lint、build 和浏览器验收。
