# V0.20A 历史 Run 重新运行实施计划

## 目标

让运行中心支持从失败、已取消或恢复失败的历史 Workflow Run 重新运行，保留原输入和版本上下文，并产生审计证据。

## 文件范围

- `apps/api/app/main.py`
- `apps/api/tests/test_execution_api.py`
- `src/api/execution.ts`
- `src/api/execution.test.ts`
- `src/pages/Runs.tsx`
- `src/pages/Runs.test.tsx`
- `src/index.css`
- `docs/CURRENT_IMPLEMENTATION.md`
- `docs/ACCEPTANCE_V0.20A.md`

## RED

1. 后端测试：对已有 Workflow Run 调用 `POST /runs/{runId}/rerun`，期望新 Run 复用 input/version 并写入 `run.rerun` 审计。
2. 前端 API 测试：`rerunWorkflowRun(workspaceId, runId)` 调用 `/api/workspaces/{workspaceId}/runs/{runId}/rerun`。
3. Runs 页面测试：失败 workflow run 展示“重新运行”，点击后提交 API，成功后选中新 Run 并提示。

## GREEN

1. 后端新增 `POST /runs/{runId}/rerun`。
2. 前端 API 新增 `rerunWorkflowRun`。
3. Runs 页面新增按钮、loading/error/success 状态和本地列表更新。
4. 样式补充操作区提示。
