# V0.22A 人工审核任务选择同步 URL 实施计划

## 目标

补齐人工审核页的任务深链闭环：`?taskId=` 已能进入并选中任务，本切片让用户在审核队列里手动切换任务后，URL 也同步更新。

## 文件范围

- `src/pages/Reviews.tsx`
- `src/pages/Reviews.test.tsx`
- `docs/ACCEPTANCE_V0.22A.md`
- `docs/CURRENT_IMPLEMENTATION.md`

## 实施步骤

1. RED：在 Reviews 测试中加入 location probe，断言点击队列任务后 URL 写入该任务 ID 并保留既有参数。
2. GREEN：把 `useSearchParams` 改为读写形式。
3. GREEN：在 `selectedId` 变化时用 `setSearchParams(..., { replace: true })` 同步 `taskId`。
4. GREEN：没有选中任务时删除 `taskId`。
5. 文档：补充 V0.22A 验收和当前实现说明。
6. 验证：聚焦测试、前后端回归、lint、build、浏览器验收。

## 验收标准

- `/reviews?taskId=...` 初始选中能力不回归。
- 点击审核队列其他任务后，URL 变更为对应 `taskId`。
- URL 中其他查询参数不丢失。
- 不改变认领、转交、通过、驳回等审核动作契约。
