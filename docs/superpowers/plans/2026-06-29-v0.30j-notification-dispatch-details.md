# V0.30J Notification Dispatch Details Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Show item-level details from the latest notification dispatch result so operators can inspect failed channels and error codes.

**Architecture:** Reuse the existing `NotificationDispatchSummary.items` returned by V0.30I. Render a compact item list under the dispatch summary, without adding backend endpoints, persisted history, or item actions.

**Tech Stack:** React, TypeScript, Vitest, Testing Library, CSS.

---

## First-Principles Gate

- Bottom-line objective: failed dispatch counts must turn into inspectable failure facts.
- Minimal required display: id, eventKey, status, channel, errorCode, providerMessageId, error.
- This is a “latest result details” view, not a durable dispatch history.

## Adversarial Review Gate

- Do not call the section “历史”.
- Do not hide errorCode or error for failed items.
- Do not add automatic retry or per-item requeue from this details list.
- Do not invent item fields not returned by the API.
- Do not create a backend query for past dispatch results.
