# V0.20B 失败节点恢复实施计划

## 目标

让失败 Workflow Run 可以从最近失败节点恢复执行，保留前序成功节点，追加新的恢复节点运行记录，并在 Runs 页面提供入口。

## 文件范围

- `apps/api/app/main.py`
- `apps/api/tests/test_execution_api.py`
- `src/api/execution.ts`
- `src/api/execution.test.ts`
- `src/pages/Runs.tsx`
- `src/pages/Runs.test.tsx`
- `docs/CURRENT_IMPLEMENTATION.md`
- `docs/ACCEPTANCE_V0.20B.md`

## RED

1. 后端测试：创建失败 Workflow Run，调用 `POST /runs/{runId}/resume-from-failed-node`，期望同一 Run 恢复完成、节点运行追加、审计事件写入。
2. 后端边界测试：Agent run 或无失败节点的 Run 调用恢复接口返回 409/422。
3. 前端 API 测试：`resumeRunFromFailedNode(workspaceId, runId)` 调用正确路径。
4. Runs 页面测试：失败 Workflow Run 显示“从失败点恢复”，点击成功后更新当前 Run 并展示提示。

## GREEN

1. 后端新增恢复路由，定位最近失败 NodeRun。
2. 调用 `ExecutionService.workflow_snapshot` 和 `execute_workflow_from(start_node_id=failed.node_id)`。
3. 写入 `run.resume_failed_node` 审计事件并提交。
4. 前端新增 API wrapper、按钮、loading/success/error 状态。
