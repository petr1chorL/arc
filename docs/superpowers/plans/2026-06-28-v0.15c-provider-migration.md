# V0.15C Provider Draft Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let operators migrate mutable Agent drafts from one model Provider to another while preserving published AgentVersion snapshots.

**Architecture:** Add a write endpoint under the source Provider that accepts `targetProviderId` and `reason`, validates the target Provider with the existing active Provider resolver, updates matching `AgentRecord` rows through the existing Provider binding helper, and records an audit event. The frontend renders a compact migration form per Provider card and refreshes Provider impact after migration.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic schemas, React, TypeScript, Vitest, Testing Library.
