# V0.14E Provider Secret Ref Runtime Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make model Provider `secretRef` participate in runtime model calls without exposing raw API keys to the frontend, database, run records, or logs.

**Architecture:** Keep raw secrets out of Agent snapshots and runtime records. `ExecutionService` resolves the bound Provider asset by `modelProviderId` and passes only its `secretRef` label into `AgentRuntimeRequest`. `OpenAICompatibleGateway` resolves the environment variable at the final outbound-call boundary and uses it as the Authorization bearer token, falling back to the global configured key when no Provider secret reference is supplied.

**Tech Stack:** FastAPI service layer, SQLAlchemy, Python dataclasses, pytest, httpx monkeypatching.

---

## File Structure

- `apps/api/tests/test_model_gateway.py`: RED/GREEN tests for resolving `model_secret_ref` without leaking the raw key.
- `apps/api/tests/test_execution_api.py`: RED/GREEN test that a Provider-bound Agent runtime passes `model_secret_ref` to the gateway.
- `apps/api/app/model_gateway.py`: accept `model_secret_ref` and resolve it from `os.environ`.
- `apps/api/app/agent_runtime.py`: carry the non-secret `model_secret_ref` label to the gateway.
- `apps/api/app/execution.py`: resolve bound Provider asset and pass its `secret_ref` label to runtime.
- `docs/ACCEPTANCE_V0.14E.md`: Chinese acceptance notes.
- `docs/CURRENT_IMPLEMENTATION.md`: update current implementation and verification evidence.
