# V0.20E Batch Resume Failed Node Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans or superpowers:test-driven-development to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Let operators select multiple failed Workflow Runs and resume each original Run from its latest failed node in one action.

**Architecture:** Add a batch endpoint that loops over requested Run IDs, reuses the existing single-run failed-node resume logic, records per-item failures instead of aborting the batch, and returns updated original Runs. The frontend extends Runs selection to expose a batch resume button for resumable failed Runs.

**Tech Stack:** FastAPI, Pydantic v2, SQLAlchemy, React, TypeScript, Testing Library, Vitest.

---
