# V0.9B Rubric 资产持久化计划

## 背景

V0.9A 已经把评估中心的反馈候选、Golden Sample 和覆盖统计切到真实 API。
但页面下方 Rubric 卡片仍来自前端 mock，导致“评分标准资产”还不能进入后续
编辑、发布、回归测试和 LLM-as-a-Judge 链路。

## 本轮目标

- [x] 新增 workspace 级 Rubric 持久化模型。
- [x] 新增 `GET /api/workspaces/{workspace_id}/evaluations/rubrics`。
- [x] 首次访问时为 workspace 播种 3 个默认 Rubric，后续访问不重复创建。
- [x] 前端评估中心从 Rubric API 加载卡片，不再读取 `src/data/mock.ts`。
- [x] 增加后端和前端测试覆盖。
- [x] 跑完整验证并记录验收证据。

## 非目标

- 不做 Rubric 新建/编辑表单。
- 不做 Rubric 发布、停用或版本冻结。
- 不做 Golden Set 管理页。
- 不做回归测试任务执行器。
- 不接入真实 LLM-as-a-Judge。

## 验收标准

- 后端 Rubric API 返回 3 个默认评分量规。
- 连续调用两次 Rubric API 不会重复播种，ID 保持稳定。
- 前端评估中心能显示 API 返回的 Rubric 名称、适用产出物、门禁、维度和通过分。
- `npm test -- src/api/evaluations.test.ts src/pages/Evaluations.test.tsx --run` 通过。
- `apps\api\.venv\Scripts\python.exe -m pytest apps\api\tests\test_evaluations_api.py -q` 通过。
- 完整 `pytest`、`vitest`、`lint`、`build` 和浏览器验证通过。
