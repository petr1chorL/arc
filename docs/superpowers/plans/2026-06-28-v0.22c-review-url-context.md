# V0.22C 人工审核 URL 上下文提示实施计划

## 目标

在人工审核页补充 URL 上下文提示，让带 `source`、`taskId`、`taskStatus` 或 `slaStatus` 的入口更容易验收和解释。

## 文件范围

- `src/pages/Reviews.tsx`
- `src/pages/Reviews.test.tsx`
- `src/index.css`
- `docs/ACCEPTANCE_V0.22C.md`
- `docs/CURRENT_IMPLEMENTATION.md`

## 实施步骤

1. RED：新增 Reviews 测试，断言带来源和筛选参数时展示“当前审核上下文”。
2. RED：断言上下文提示列出来源、任务 ID、任务状态和 SLA。
3. RED：断言点击“清空上下文筛选”会清空任务状态和 SLA 参数。
4. GREEN：在 Reviews 中读取 `source` 并生成来源文案。
5. GREEN：在概览下方渲染上下文提示和清空按钮。
6. GREEN：补充 CSS，确保提示条在桌面和移动端不挤压文本。
7. 文档：补充验收文档和当前实现说明。
8. 验证：聚焦测试、全量回归、lint、build、浏览器验收。

## 验收标准

- URL 上下文提示只在有来源或非默认筛选时出现。
- 提示内容能解释来源、当前任务和筛选。
- 清空按钮只清空状态和 SLA 筛选，不丢失 `taskId` 和来源参数。
- 不改变人工审核动作和权限契约。
