# V0.13N Execution Queue Status Filter Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development to implement this plan task-by-task.

**Goal:** Let operators filter the Observability execution queue by job status.

**Architecture:** Reuse the existing `listExecutionJobs(workspaceId, status?)` API wrapper. Keep the filter local to the Observability queue panel, because it is an operational lens rather than a global run filter. Clear the inline job detail when the queue filter changes so stale detail from a hidden job is not shown.

**Tech Stack:** React, TypeScript, Vitest, Testing Library, native CSS.
