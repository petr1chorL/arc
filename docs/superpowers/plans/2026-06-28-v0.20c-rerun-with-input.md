# V0.20C 带输入覆盖的历史 Run 重新运行实施计划

## 目标

让运行中心的历史 Workflow Run 重新运行支持可选输入覆盖。默认行为不变；当用户修改输入后重跑，新 Run 使用覆盖输入，并在审计 metadata 中记录覆盖事实。

## 文件范围

- `apps/api/app/schemas.py`
- `apps/api/app/main.py`
- `apps/api/tests/test_execution_api.py`
- `src/api/execution.ts`
- `src/api/execution.test.ts`
- `src/pages/Runs.tsx`
- `src/pages/Runs.test.tsx`
- `docs/CURRENT_IMPLEMENTATION.md`
- `docs/ACCEPTANCE_V0.20C.md`

## RED

1. 后端测试：`POST /runs/{runId}/rerun` 传入覆盖输入后，新 Run 使用覆盖输入，审计 `inputOverridden=true`。
2. 后端测试：空白覆盖输入返回 `422`。
3. 前端 API 测试：`rerunWorkflowRun(workspaceId, runId, { input })` 会发送 JSON body。
4. Runs 页面测试：点击“编辑输入重跑”，修改输入，确认后选中新 Run。

## GREEN

1. 新增 `RunRerunRequest`，`input` 可选但提供时必须非空。
2. rerun endpoint 根据 payload 选择源输入或覆盖输入。
3. 审计 metadata 增加 `inputOverridden`。
4. 前端 API wrapper 支持可选 input。
5. Runs 页面增加紧凑的输入覆盖区、取消和确认动作。
