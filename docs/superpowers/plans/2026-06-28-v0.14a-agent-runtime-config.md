# V0.14A Agent Runtime Config Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development to implement this plan task-by-task.

**Goal:** Add the first persisted Agent-level runtime configuration entry point without storing provider secrets.

**Architecture:** Extend the existing Agent draft contract rather than adding a new provider asset table yet. Persist non-secret runtime fields on `agents`, include them in `AgentRead`, and let `agent_snapshot()` publish them through the existing immutable snapshot path. Frontend edits these fields from Agent detail and submits them with the existing draft update call.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic, SQLite migration helper, React, TypeScript, Vitest, Testing Library, native CSS.
