# V0.30B Notification Outbox Worker Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a CLI and worker entrypoint that can consume Notification Outbox rows without manually calling the HTTP dispatch API.

**Architecture:** Create a focused `app.notification_worker` module parallel to the existing `app.worker` execution queue worker. The worker reuses the V0.30A `NotificationOutboxDispatchService`, iterates active Workspaces, commits dispatch results, and exposes `--once` / `--until-idle` / forever modes. Add a console script and Compose service definition.

**Tech Stack:** Python argparse, FastAPI app factory, SQLAlchemy session factory, Pytest, Docker Compose config.
