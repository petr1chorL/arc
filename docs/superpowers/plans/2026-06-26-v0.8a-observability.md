# V0.8A Observability Implementation Plan

## Goal

Build the first lightweight observability slice: a Workspace-scoped operations page that summarizes runs, highlights failures and human-review blockers, and shows node-level troubleshooting detail.

## Architecture

Use existing FastAPI + SQLAlchemy records and React state. Do not add external observability infrastructure yet.

## Task 1: Backend Observability API

### Step 1: RED backend tests

Create `apps/api/tests/test_observability_api.py`.

Tests:

- `GET /api/workspaces/{workspace_id}/observability/overview` returns totals and recent runs.
- Failed and waiting-for-human runs are prioritized before completed runs.
- `GET /api/workspaces/{workspace_id}/observability/runs/{run_id}` returns node runs and related Human Tasks.
- Cross-Workspace run detail returns 404.

Command:

```powershell
apps\api\.venv\Scripts\python.exe -m pytest apps\api\tests\test_observability_api.py -q
```
