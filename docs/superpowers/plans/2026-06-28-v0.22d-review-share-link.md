# V0.22D 人工审核上下文复制链接实施计划

## 目标

在 V0.22C 的“当前审核上下文”提示中增加一键复制当前链接，让人工审核任务和筛选上下文更容易交接、验收和排障。

## 文件范围

- `src/pages/Reviews.tsx`
- `src/pages/Reviews.test.tsx`
- `src/index.css`
- `docs/ACCEPTANCE_V0.22D.md`
- `docs/CURRENT_IMPLEMENTATION.md`

## 实施步骤

1. RED：在 Reviews 测试中 mock `navigator.clipboard.writeText`，断言点击“复制当前链接”会写入当前完整 URL。
2. RED：新增失败路径测试，剪贴板写入失败时显示“复制失败，请手动复制地址栏链接”。
3. GREEN：在 Reviews 页面增加复制状态和 `copyReviewLink` 处理函数。
4. GREEN：在“当前审核上下文”提示中增加复制按钮和成功/失败提示。
5. GREEN：补充上下文操作区样式，保证桌面和窄屏不挤压文本。
6. 文档：新增 V0.22D 验收文档，更新当前实现说明。
7. 验证：聚焦 Reviews 测试、前端全量测试、后端测试、lint、build、浏览器验收。

## 验收标准

- 带 URL 上下文进入人工审核页时可以看到“复制当前链接”。
- 点击复制按钮后写入当前完整 URL。
- 成功和失败提示清晰可见。
- 不新增后端接口，不改变人工审核动作、Reviewer 权限或 URL 同步规则。
