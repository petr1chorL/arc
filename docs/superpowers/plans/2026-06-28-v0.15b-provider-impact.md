# V0.15B Provider Impact View Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Show which Agent drafts and published Agent versions depend on each model Provider.

**Architecture:** Add a read-only Workspace-scoped Provider impact endpoint that derives draft dependencies from `AgentRecord.model_provider_id` and published dependencies from `AgentVersionRecord.snapshot.modelProviderId`. The frontend fetches impact summaries alongside the Provider list and renders compact dependency counts on each Provider card.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic schemas, React, TypeScript, Vitest, Testing Library.
