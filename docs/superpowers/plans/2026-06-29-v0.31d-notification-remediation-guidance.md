# V0.31D Notification Remediation Guidance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Translate notification dispatch failure codes into actionable troubleshooting guidance on the Notification Ops page.

**Architecture:** Frontend-only deterministic mapping in `Notifications.tsx`, with tests. No backend endpoints or real external delivery.

**Tech Stack:** React, TypeScript, React Router, Vitest, Testing Library, CSS.

---

## First-Principles Gate

- Bottom-line objective: failure evidence must lead to a next operational action.
- Minimal objects: error code, channel, advice text, current Workspace settings link.
- This is guidance, not remediation automation.

## Adversarial Review Gate

- Do not hide original error codes.
- Do not imply automatic repair.
- Do not blame user configuration when adapter code is missing.
- Do not link outside the current Workspace.
