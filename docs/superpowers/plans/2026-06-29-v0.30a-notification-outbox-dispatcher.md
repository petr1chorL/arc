# V0.30A Notification Outbox Dispatcher Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the first backend slice that consumes pending Notification Outbox rows through an injectable dispatcher and records send results.

**Architecture:** Keep `notification_outbox` as the source table and avoid schema changes in this slice. Add a small `app.notification_dispatcher` module with delivery/result DTOs, a default no-op dispatcher, and a service that dispatches pending rows for one Workspace. Expose a Workspace API route that applies existing auth, CSRF, and RBAC rules.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic, Pytest, existing ARC.ONE auth/RBAC services.
