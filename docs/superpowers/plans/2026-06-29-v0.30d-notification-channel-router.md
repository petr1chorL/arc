# V0.30D Notification Channel Router Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a local Notification Channel Router boundary so Outbox dispatch can route deliveries to channel adapters without connecting real external services.

**Architecture:** Extend `app.notification_dispatcher` with a small router that implements the existing `NotificationDispatcher` protocol. The router reads channel intent from `NotificationDelivery.payload`, delegates to enabled adapters, and returns structured failed dispatch results for unknown or disabled channels.

**Tech Stack:** FastAPI app factory, SQLAlchemy-backed Outbox records, Python dataclasses/Protocol, Pytest.

---

## First-Principles Gate

- Bottom-line objective: a business event must be deliverable through an explicit target channel.
- Minimal objects: delivery payload channel intent, channel adapter registry, enabled/disabled channel state, structured dispatch result.
- This precedes real Feishu/email/Webhook integration because credentials, rate limits, templates, callbacks, and idempotency are separate production concerns.

## Adversarial Review Gate

- Do not let unknown channels crash the worker; failed outbox evidence is better than a dead process.
- Do not silently fall back from an explicit unknown channel to `in_app`; that would create false delivery confidence.
- Do not call disabled adapters.
- Do not claim real external notifications exist; default adapters remain local/noop.
- Do not add schema/table churn until the routing boundary proves useful.

## Task 2: Implement Router

**Files:**
- Modify: `apps/api/app/notification_dispatcher.py`
- Modify: `apps/api/app/main.py`

- [x] **Step 1: Add adapter dataclass and router**

Implement:

- `NotificationChannelAdapter(name, dispatcher, enabled=True)`.
- `NotificationChannelRouter(adapters, default_channel="in_app")`.
- channel resolution from `payload.channel`, then first entry of `payload.channels`, then default.
- unknown channel returns `NotificationDispatchResult(status="failed", error="channel_not_configured:<channel>")`.
- disabled channel returns `NotificationDispatchResult(status="failed", error="channel_disabled:<channel>")`.
