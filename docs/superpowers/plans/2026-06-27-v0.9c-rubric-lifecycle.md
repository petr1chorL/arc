# V0.9C Rubric 生命周期实施计划

## 背景

V0.9B 已经把评分量规从前端 mock 迁移到 workspace 级后端资产，但只能浏览，不能创建、编辑、发布不可变版本或停用。V0.9C 补齐评分标准资产的最小闭环。

## 范围

- 新建评分量规草稿。
- 编辑草稿或已启用量规的职责字段：名称、适用产出物、维度、权重、硬性门禁、通过分数。
- 发布不可变 Rubric 版本。
- 查看已发布版本快照。
- 停用评分量规，停用后不允许继续编辑或发布。
- 前端提供必填校验、权重合计校验和错误提示。

## 不做

- 不接入真实评估执行器。
- 不做 Rubric 模板市场。
- 不做发布审批流。
- 不做版本回滚。

## 实施步骤

1. 先写后端 API 红灯测试，覆盖创建、编辑、发布、版本快照冻结、停用和非法输入。
2. 写前端 API 客户端红灯测试，约束请求路径和方法。
3. 写评估中心页面红灯测试，约束新建弹窗和客户端权重校验。
4. 新增 Rubric 版本表和生命周期 API。
5. 前端评估中心接入新建、编辑、发布、停用和版本列表。
6. 更新当前实现文档和 V0.9C 验收文档。
7. 跑 API、前端单测、lint、build 和浏览器验收。

## 验收标准

- `POST /evaluations/rubrics` 能创建 draft Rubric。
- `PATCH /evaluations/rubrics/{id}` 能编辑非停用 Rubric。
- `POST /evaluations/rubrics/{id}/publish` 返回不可变版本快照。
- 发布后再编辑 Rubric，不会改变已发布版本快照。
- `POST /evaluations/rubrics/{id}/deactivate` 后，继续编辑返回 409。
- 前端新建量规时，维度权重合计不是 100 会提示“维度权重合计必须等于 100”。
- 前端能保存后立即看到新 Rubric 卡片。
