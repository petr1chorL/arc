# V0.14C Agent Provider Binding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let Agent drafts bind to a Workspace model Provider asset and publish that binding into immutable Agent versions.

**Architecture:** Keep Provider credentials out of Agent records. Store a nullable `model_provider_id` on Agent drafts, validate it against the current Workspace, and denormalize provider type/Base URL/default model into existing runtime config fields for compatibility. Frontend Agent details should load Provider assets and offer a select control instead of free-form provider text.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic, SQLite migration helper, React, TypeScript, Vitest, Testing Library.

---

- `apps/api/app/models.py`: add `AgentRecord.model_provider_id`.

- `docs/ACCEPTANCE_V0.14C.md`: Chinese acceptance notes.
