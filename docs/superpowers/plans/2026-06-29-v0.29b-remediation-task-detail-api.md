# V0.29B Remediation Task Detail API Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a stable single Remediation Task detail read path so `taskId` deep links work even when the task is not in the current list response.

**Architecture:** FastAPI adds a read-only `GET /evaluations/remediation-tasks/{taskId}` endpoint that reuses the existing `RemediationTaskRead` projection. React adds a `getRemediationTask` API client and a deep-link lookup effect that merges the fetched task into local state without resetting current filters.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic, React, TypeScript, Vitest, Testing Library, Pytest.
