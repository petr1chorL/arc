# V0.11B Remediation Activity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add comments and processing timeline records to Remediation Tasks.

**Architecture:** Add a dedicated `RemediationTaskActivityRecord` table and expose activity records through task reads. Status changes create system activity records; comment creation uses a new task-scoped endpoint. The frontend renders a compact timeline inside each task card and posts comment records through the existing evaluation API client.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic, SQLite migration helper, React, TypeScript, Vitest, Pytest.
