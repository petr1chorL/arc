# V0.24A Workflow IO Schema Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add editable workflow-level input and output JSON Schema drafts that persist and freeze into workflow versions.

**Architecture:** Store `input_schema` and `output_schema` JSON columns on `WorkflowRecord`, expose them through `WorkflowCreate`, `WorkflowUpdate`, and `WorkflowRead`, and include them in the existing `workflow_snapshot`. The Workflows page keeps JSON text state, validates that each value parses to an object before save, and sends `inputSchema` / `outputSchema` with the existing draft save request.

**Tech Stack:** FastAPI, SQLAlchemy JSON columns, Pydantic aliases, React 19, TypeScript, Vitest, Testing Library, Pytest.
