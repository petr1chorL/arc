# V0.23A 工作流节点库拖拽入画布实施计划

## 目标

补齐工作流编排中心的节点库拖拽能力，让用户可以把节点从左侧节点库拖到画布落点创建。

## 文件范围

- `src/pages/Workflows.tsx`
- `src/pages/Workflows.test.tsx`
- `src/index.css`
- `docs/ACCEPTANCE_V0.23A.md`
- `docs/CURRENT_IMPLEMENTATION.md`

## 实施步骤

1. RED：扩展 Workflows 测试里的 ReactFlow mock，暴露一个可 drop 的画布区域，并传入 `screenToFlowPosition`。
2. RED：新增测试，拖拽“人工审核”节点到画布，断言新增节点出现且保存请求体包含落点位置。
3. GREEN：为节点库项增加 `draggable`、`dragstart` 和键值 payload。
4. GREEN：在画布容器处理 `dragover/drop`，读取 payload 并用 React Flow instance 的 `screenToFlowPosition` 计算位置。
5. GREEN：复用现有 `addNode` 数据生成逻辑，支持点击默认位置和拖拽指定位置两条路径。
6. 文档：新增验收文档，更新当前实现说明。
7. 验证：聚焦 Workflows 测试、前端全量、后端全量、lint、build、浏览器验收。

## 验收标准

- 节点库项可拖拽。
- 拖入画布后新增对应节点，位置来自释放坐标。
- 点击添加、连线、保存草稿不回归。
- 不新增后端接口，不改变工作流草稿契约。
