# V0.14F Provider Lifecycle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add model Provider edit and deactivate workflows, and prevent disabled Providers from being newly bound to Agents.

**Architecture:** Keep Provider lifecycle Workspace-scoped. Add PATCH and deactivate endpoints that update only non-secret metadata plus `secretRef` label. Agent binding already resolves Provider assets; extend that resolver to reject disabled Providers. Frontend Provider cards get lightweight inline editing and a deactivate action without ever exposing raw keys.

**Tech Stack:** FastAPI, SQLAlchemy, Pydantic, React, TypeScript, Vitest, Testing Library.
