# V0.8D Trace Skeleton Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a lightweight Trace ID and Span skeleton to workflow runs so the observability page can show one run-level trace and node/audit event linkage.

**Architecture:** Keep the implementation inside the existing FastAPI + SQLAlchemy + React app. Store `trace_id` on workflow runs, `trace_id`/`span_id`/`parent_span_id` on node runs, and `trace_id`/`span_id` on audit events; lazily backfill missing values when observability detail is read so old SQLite data remains usable. Do not add OpenTelemetry, Tempo, Langfuse, workers, or external tracing services in this slice.

**Tech Stack:** FastAPI, SQLAlchemy, SQLite lightweight migrations, Pydantic, React, Vitest, Testing Library, Pytest.
