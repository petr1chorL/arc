# V0.10D Regression Run 趋势视图验收

## 验收目标

确认评估中心能基于多次 Regression Run 展示质量趋势，帮助判断最近回归是否稳定、是否出现通过率下滑。

## 前置条件

- 已登录本地 ARC.ONE。
- 当前 Workspace 至少有 2 次 Regression Run。
- 如果 Run 不足 2 次，可在评估中心使用同一个 Rubric 连续运行两次批量回归。

## 验收步骤

1. 打开 `http://127.0.0.1:4173/w/ai-capability-center/evaluations`。
2. 滚动到 `Regression Run History`。
3. 确认出现 `Regression Run Trend` 区块。
4. 确认趋势区展示：
   - 最新通过率。
   - 较上次变化。
   - 平均通过率。
   - 最佳通过率。
   - 当前参与趋势计算的 Run 数量。
5. 确认趋势柱状图按时间从旧到新展示最近 Run。
6. 使用 Rubric 或 Run 状态筛选后，确认趋势指标会随筛选结果刷新。
7. 如果存在低于 70% 的 Run，确认对应趋势点显示风险标记。

## 不通过判定

- 有 2 次以上 Run 时仍不显示趋势区。
- 趋势指标与 Run 历史列表明显不一致。
- 切换筛选后趋势区不刷新。
- 低通过率 Run 没有风险标记。
- 原有 Run 历史列表、详情弹窗或 Run 对比不可用。

## 当前限制

- V0.10D 的趋势计算在前端完成，暂未提供后端聚合 API。
- 趋势最多展示最近 8 次 Run。
- 暂不包含导出报告、阈值配置和跨 Rubric 聚合分析。
