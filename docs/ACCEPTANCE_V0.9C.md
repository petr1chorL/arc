# V0.9C 评分量规生命周期验收

## 这版做了什么

V0.9C 把评分量规从“只能看”推进到“可管理资产”：

1. 新增 `rubric_versions` 数据表，用于保存发布后的不可变 Rubric 快照。
2. 新增 Rubric 创建、编辑、发布版本、查看版本、停用 API。
3. 新增 Rubric 写入和发布权限能力：`rubric.write`、`rubric.publish`。
4. 评估中心增加“新建评分量规”弹窗。
5. 每张 Rubric 卡片可进入配置弹窗，支持保存、发布版本、停用。
6. 前端增加必填校验、通过分数范围校验、维度权重合计必须等于 100 的校验。

## 人工验收路径

打开：

```text
http://127.0.0.1:4173/w/ai-capability-center/evaluations
```

### 1. 新建校验

1. 点击“新建评分量规”。
2. 输入名称、适用产出物、硬性门禁、通过分数。
3. 把“维度 1 权重”改成 `80`。
4. 点击“保存评分量规”。
5. 应看到错误提示：`维度权重合计必须等于 100`。

### 2. 新建成功

1. 把“维度 1 权重”改回 `100`。
2. 点击“保存评分量规”。
3. 页面应出现新 Rubric 卡片。
4. 卡片中应能看到名称、适用产出物、硬性门禁、维度和通过分数。

### 3. 发布版本

1. 点击某张 Rubric 卡片右上角配置按钮。
2. 点击“发布版本”。
3. 应显示“已发布不可变版本 v1.0.0”。
4. 弹窗底部的不可变版本列表应出现该版本。

### 4. 版本不可变

1. 发布后修改通过分数并保存。
2. 再查看版本列表。
3. 已发布版本里的快照分数不应被这次编辑改变。

### 5. 停用

1. 点击“停用”。
2. 弹窗应提示“评分量规已停用”。
3. 保存和发布按钮应不可用。
