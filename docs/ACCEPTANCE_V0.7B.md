# V0.7B 人工审核闭环验收清单

## 验收目标

确认你可以从账号资格配置开始，完整跑通：

```text
登录
-> 绑定当前账号 Reviewer 资格
-> 运行包含 Human 节点的工作流
-> 在人工审核页认领任务
-> 提交审核决定
-> 查看运行恢复、审计记录和反馈候选
```

## 1. 打开系统

1. 启动本地服务。
2. 打开 `http://127.0.0.1:4173`。
3. 使用默认管理员账号进入 `AI 能力中心` Workspace。

验收通过标准：

- 左侧导航、顶部账号和 Workspace 名称正常显示。
- 进入 `人工审核` 页面不出现空白页或控制台错误。

## 2. 绑定当前账号 Reviewer 资格

1. 进入 `成员与权限`。
2. 找到页面顶部的 `当前账号审核资格` 面板。
3. 如果显示 `当前账号未获得 Reviewer 资格`，保留默认 `产品审核人`，点击 `绑定当前账号为 Reviewer`。
4. 回到 `人工审核`。

验收通过标准：

- 成员页出现 `当前账号已绑定 Reviewer 资格。`。
- 人工审核页 `当前用户` 区域不再显示 `未获得 Reviewer 资格`。
- `认领任务`、`通过`、`驳回` 等按钮的禁用原因能被页面解释。

## 3. 产生人工审核任务

1. 进入 `工作流`。
2. 选择或创建一个包含 `人工审核` Human 节点的工作流。
3. 确认 Human 节点已配置审核人或审核组。
4. 发布工作流版本。
5. 运行工作流。

验收通过标准：

- 运行结果进入 `需介入` 状态。
- 运行弹窗提供进入 `人工审核` 和 `运行中心` 的入口。
- `运行中心` 对 `需介入` 运行显示等待人工审核提示。

## 4. 处理人工审核任务

1. 进入 `人工审核`。
2. 选择任务。
3. 查看 `当前任务权限`。
4. 点击 `认领任务`。
5. 填写 `审核原因`。
6. 选择 `通过`、`驳回`、`退回重跑` 或 `修改后通过`。

验收通过标准：

- 未填写审核原因时会提示必填。
- 提交决定后任务状态更新。
- 任务上下文中能看到审计时间线。
- 导航角标会随未终结 Human Task 数量变化。

## 5. 验证修改后通过与反馈候选

1. 在人工审核任务中点击 `编辑产出物`。
2. 修改 Artifact 内容。
3. 填写审核原因。
4. 点击 `修改后通过`。

验收通过标准：

- 系统创建新的 Artifact 版本。
- 任务上下文显示 Diff。
- 产生 FeedbackCandidate。
- 质量专家账号可以确认 Golden Sample。

## 6. 无任务时的验收诊断

当人工审核页没有 Human Task 时，检查空状态。

验收通过标准：

- 页面显示 `暂无人工任务`。
- 页面展示 3 步验收路径。
- 页面展示 `验收诊断`，包含当前账号、Reviewer 资格、人工任务数量、最近运行状态和下一步建议。
- 页面提供 `去工作流编排` 和 `查看成员与权限` 入口。

## 当前限制

- Human Task 不会由人工审核页手动创建，只能由工作流运行到 Human 节点时产生。
- Reviewer/ReviewGroup 仍是本地平台内管理，未接入企业通讯录。
- 飞书等外部通知尚未发送，只写入 NotificationOutbox。
- 运行恢复、审计记录和反馈候选已持久化，但还不是完整生产级观测链路。
