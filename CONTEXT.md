# ARC.ONE 领域语言

ARC.ONE 的领域语言用于准确描述企业 Agent 工作流的设计、治理与运行。

## 领域语言（Language）

**业务场景（Business Scenario）**：
具有明确边界、负责人、预期成果和衡量标准的业务问题。
_避免使用_：用例、项目、应用

**工作流（Workflow）**：
为实现业务成果而协调多个节点运行的有向流程。
_避免使用_：流程链、流水线、Agent 链

**工作流版本（Workflow Version）**：
工作流及其依赖资产版本的不可变快照。
_避免使用_：副本、修订、备份

**节点（Node）**：
工作流中承担一次执行、判断、评估或人工工作的基本单元。
_避免使用_：步骤、积木、组件

**Agent**：
具有明确职责、上下文、工具、知识、契约、限制和评估要求的版本化智能能力。
_避免使用_：机器人、Prompt、助手

**技能（Skill）**：
Agent 可通过明确契约调用的可复用业务能力。
_避免使用_：函数、插件、技巧

**工具（Tool）**：
Agent 读取数据或对外部系统产生操作的受控接口。
_避免使用_：技能、连接器、API

**数据对象（Data Object）**：
节点之间交换信息时采用的版本化结构定义。
_避免使用_：Payload、JSON、消息

**产出物（Artifact）**：
由节点运行产生、符合数据对象定义且可追溯的具体成果。
_避免使用_：结果、回复、文件

**评分量规（Rubric）**：
由评分维度、权重、评分锚点和通过阈值构成的版本化质量定义。
_避免使用_：打分表、检查表、评分 Prompt

**质量门禁（Quality Gate）**：
决定产出物继续流转、重试、停止或进入人工审核的准入判断。
_避免使用_：评分量规、校验、分数

**黄金样本（Golden Sample）**：
由专家确认的输入与期望结果组合，用于评估和回归测试。
_避免使用_：演示、普通示例、测试夹具

**评估（Evaluation）**：
对产出物、节点运行或工作流运行进行的独立质量判断。
_避免使用_：自评、分数、测试

**工作流运行（Workflow Run）**：
某一工作流版本从触发到终态的一次完整执行。
_避免使用_：Job、任务、Session

**节点运行（Node Run）**：
某个节点在一次工作流运行中的一次执行尝试。
_避免使用_：步骤结果、调用、Trace

**人工任务（Human Task）**：
在规定服务时限内分配给人员并要求记录决策的正式节点任务。
_避免使用_：人工异常、线下审核、审批消息

**人工审核（Human Review）**：
人员通过人工任务评价或修改产出物的行为。
_避免使用_：人工修补、临时介入

**反馈（Feedback）**：
被记录的修正或结果信号，可进一步成为评估证据或黄金样本。
_避免使用_：评论、备注、个人意见

**质量结果（Quality Result）**：
质量门禁的完整输出，包括门禁状态、分数、证据和路由决定。
_避免使用_：单一分数、通过率
